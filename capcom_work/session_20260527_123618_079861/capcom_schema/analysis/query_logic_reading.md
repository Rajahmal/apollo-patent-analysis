# 母集団検索式（query_logic）の読解と整合性検査ガイド

> このファイル名はレポート本文に書かないこと（執筆者の内部参照専用）。
> 用語統一ルール → `analysis/terminology.md`（§5-A `query_intent` の 3 原則、§6 スコープ限定ルール）
> 品質検証 → `analysis/quality_checklist.md`

---

## 0. 本ガイドの目的

APOLLO v8 では、ユーザーが CAPCOM ページで入力した **`population_meta.query_logic`（母集団検索式）** を付録 D に全文掲載するだけでなく、Phase A で執筆者（エージェント）が **構造化して読解・ユーザー確認・データ整合性検査** を行うことを義務化する。

この 3 ステップ（A / B / C）は Phase A の中核であり、それぞれ独立した STOP-GATE を構成する：

| ステップ | 名称 | 目的 |
|---|---|---|
| **A** | `query_logic` 構造化読解 | 検索式を論理要素に分解し、どんな母集団が抽出されているかを把握する |
| **B** | 意図 ↔ 論理 整合性検査 | `query_intent`（設計意図）と `query_logic`（実装）の乖離を検出し、ユーザー確認 |
| **C** | データ側からの逆読み | patents.csv から実際に得られた母集団の統計特徴を確認し、設計との乖離を検出 |

3 段階とも `AskUserQuestion` で STOP-GATE を構成し、ユーザー応答を得るまで次に進まない。

本ガイドは、エージェントがこれら 3 ステップを実施するために必要な **DB 別構文リファレンス**、**共通原則**、**乖離検出チェックリスト** を集約したものである。

---

## 1. 読解の一般プロセス（DB 非依存の 4 ステップ）

`query_logic` を読解する際、以下の 4 ステップを順に実行する：

### Step 1: DB 識別

- まず `population_meta.database_name` を確認する。値があればそれを使う
- 値がなければ、検索式のフォーマットから推測する（次節「DB 別構文リファレンス」の構文特徴を参照）
- 識別できない場合、`AskUserQuestion` でユーザーに「この検索式はどのデータベースのものですか?」と確認する

### Step 2: 構文分解

- 論理演算子（AND / OR / NOT）で式を節に分ける
- 各節を以下のカテゴリに仕分ける：
  - **分類条件**（IPC / FI / Fターム / CPC 等）
  - **キーワード条件**（全文 / 要約 / 請求項 / タイトル 等）
  - **出願人条件**（PA / assignee / applicant 等）
  - **日付条件**（出願日 / 公開日 / 優先日 等）
  - **その他**（国指定 / 公報種別 / 審査状態 等）

### Step 3: 各条件の意図推定

- 各条件について「なぜこの条件がここにあるか」を推定する
- 例:
  - `IC=C08L*/IC` → 「高分子組成物関連を広く拾う」
  - `NOT A23*/IC` → 「食品分野を除外」
  - `AD=20200101:20251231` → 「直近 6 年に限定して最新動向に焦点」
  - `AND PA=(旭化成+東レ+帝人)` → 「主要 3 社に絞る」

### Step 4: ユーザー確認（STOP-GATE A）

- Step 1-3 をまとめて `AskUserQuestion` で提示
- ユーザーが「この読解で合っている / 修正が必要 / 補足を追加」のいずれかを選択するまで次に進まない

## 2. DB 別構文リファレンス

以下は 2026-04-19 時点で確認できた公開情報に基づく構文サマリ。**公開情報で断定できない部分は「確認要」と明示**し、実機マニュアルや契約者向けドキュメントでの最終確認を推奨する。

### 2-1. J-PlatPat（INPIT 公式・無償）

**提供元**: 独立行政法人工業所有権情報・研修館（INPIT）／特許庁系
**位置づけ**: 無償公開サービス。論理式入力は UI と並行して利用可能

#### 演算子
| 種別 | 記号 | 優先順位 |
|---|---|---|
| AND | `*` | 2 |
| OR | `+` | 3 |
| NOT | `-` | 3 |

#### 括弧
- `[ ]` 大括弧（優先順位 1、**3 重入れ子まで**）
- `( )` 丸括弧（入力省略用）

#### 近傍演算子
| 記法 | 意味 |
|---|---|
| `,{n}C,` | **語順固定**（C = Controlled）、n は 1-99 |
| `,{n}N,` | **語順不問**（N = Neutral）、n は 1-99 |

**距離単位**: 日本語 = 文字数、英語 = 単語数
**3 単語近傍**: `{A,5C,B},3N,C`（中括弧で優先度指定。省略形は **N/n のみ**可、C は不可）

#### フィールド指定タグ（末尾に `/` + タグ）
| タグ | 内容 |
|---|---|
| `/TX` | 全文 |
| `/AB` | 要約 |
| `/CL` | 請求の範囲 |
| `/TI` | タイトル |
| `/FI` | FI 分類 |
| `/FT` | F ターム |
| `/IP` | IPC |
| `/CP` | CPC |
| `/PN` | 優先権主張番号 |
| `/AN` | 出願人識別番号 |

#### ワイルドカード
- `?` 任意 1 文字（和文・英文両対応）
- `*` 0 文字以上の任意文字列（**英文のみ**）

#### 制限
- 論理式全体: **500 文字**
- キーワード: 全角 20 文字 / 半角 40 文字

#### 例
```
[クラウド/AB+システム/AB]*[製造方法/CL]
[無電源,5C,発光/TX]
[フード,5N,ダクト/TI]
```

#### 識別用の構文特徴
- タグが末尾に `/TX` `/AB` 等の形式
- 近傍演算子が `,{n}C,` / `,{n}N,` 形式
- 大括弧 `[ ]` が使われる

**Sources**:
- [J-PlatPat 論理式入力ヘルプ（公式）](https://www.j-platpat.inpit.go.jp/help/ja/p01/arithmetic.html)
- [近傍検索の活用方法（秋山国際特許商標事務所）](https://www.tectra.jp/akiyama-patent/post-1326/)

### 2-2. JP-NET（JPDS・商用）

**提供元**: 日本パテントデータサービス株式会社（JPDS, Japan Patent Data Service）
**位置づけ**: **商用有償サービス**。**特許庁系ではない**。J-PlatPat とは提供元も構文も機能も全く異なる（名前が似ているが別物）

#### 演算子
| 種別 | 記号 |
|---|---|
| AND | `&`（または `AND`） |
| OR | `+`（または `OR`） |
| NOT | `!`（または `NOT`） |

#### 括弧
- `( )` 丸括弧

#### 近傍演算子
| 記法 | 意味 |
|---|---|
| `<{n}W>` | **語順固定**、n は 1-99 |
| `[{n}W]` | **語順不問**、n は 1-99 |
| `&` | **同一センテンス内**（順序不問） |

**完全一致 / 前後一致**: `||語||`（完全）/ `語||`（前方一致）/ `||語`（後方一致）

**併用不可**: `<>` と `&` は併用不可（括弧で明示すれば可）、`<nW>` と `[nW]` も同様

**3 単語グループまで**（順不同検索時）、4 つ以上はエラー

#### フィールド指定コマンド（行頭に `コマンド=`）
| コマンド | 内容 |
|---|---|
| `HTX` | 全文の語句 |
| `HTI` | タイトルの語句 |
| `HAB` | 要約の語句 |
| `HCL` | クレームの語句 |
| `HTC` | タイトル〜クレーム |
| `HO2` | 技術分野 |
| `HBA` | 背景技術 |
| `HPR` | 課題 |
| `HO3` | 解決手段 |
| `HDI` | 実施例 |
| `HEF` | 発明の効果 |
| `UTX` 系 | 出願経過系（全 11 種） |
| `IC` | IPC・展開記号（`IC=C08L51/00` など） |
| `FI` | FI |
| `FTM` | F ターム |
| `PA` | 出願人 |
| `IN` | 発明者 |
| `AD` | 出願日 |
| `PD` | 公開日 |
| `T` | 結合コマンド（集合演算） |

#### 分類検索の階層指定
- `IC=A01B1/00.` ドット階層（A01B1/00 とその下位階層）
- `IC=A01B1/4` スラッシュ以下すべて
- `IC=A01B1/4*A01B3/x` 範囲指定（`*` は「以下」の意）

#### 集合演算
- `T=#1&#2` → 式 #1 AND 式 #2
- `T=#1+#2` → 式 #1 OR 式 #2
- `T=(#1+#2)&#3` → (#1 OR #2) AND #3
- `T=#1-#3` → #1 から #3 までの範囲（連続式を一括）
- `T=#1!#2` → NOT 演算

#### ワイルドカード
- `?` 任意 1 文字
- `*` トランケーションではなく AND 記号なので注意

#### 制限
- 1 行 150 バイト、改行継続可（`+Enter`）
- 最大 #200 まで、改行で最大 2000 行
- 英数字は大文字入力（小文字は Shift 併用）

#### 例
```
HTX=自動運転
PA=トヨタ自動車
IC=H01L27/10
T=#1&#2&#3

UTX=自動走行<10W>車           // 10 文字以内、順序固定
UTX=自動走行[10W]車           // 10 文字以内、順序不問
UTX=||自動走行|| & ||車||     // 完全一致
```

#### 識別用の構文特徴
- 行頭に `HTX=` `HAB=` `IC=` `PA=` 等のコマンド
- 近傍演算子が `<nW>` / `[nW]` 形式
- `T=#1&#2` 形式の集合演算

### 2-3. Patentfield（日本商用）

**提供元**: Patentfield 株式会社
**位置づけ**: コマンド検索が公開されている商用 DB。**日本語近傍検索が特に強い**

#### 演算子
| 種別 | 記号 |
|---|---|
| AND | `and` / `+` / 省略時デフォルト |
| OR | `OR` |
| NOT | `not` / `-` |

#### 括弧
- `( )` 丸括弧

#### 近傍演算子（トークン単位）
| 記法 | 意味 |
|---|---|
| `*N{n}"語1 語2"` | **順不問**、n 距離以内 |
| `*NP{n}"フレーズ1 フレーズ2"` | 順不問（N と同結果だがフレーズ）|
| `*ONP{n}"フレーズ1 フレーズ2"` | **指定順**近傍 |

**距離単位**: 日本語 = 文字数、英語 = 単語数
**特殊指定**:
- 同一センテンス指定: `。|` を末尾に追加
- オーバーラップ抑制: `,>0`
- 複数距離: `*ONP,5|3"太陽 発電 パネル"`

**⚠️ 制限**: 近傍式の内部は **入れ子括弧および AND/OR/NOT をサポートしない**（B の整合性検査で要注意）

#### フィールド指定（先頭に `コード:`）
| コード | 内容 |
|---|---|
| `KW` | キーワード（名称＋要約＋請求範囲＋明細書＋審査官フリーワード、**デフォルト**） |
| `CL` | 請求の範囲 |
| `IPC` | IPC 分類 |
| `AD` | 出願日 |
| `AN` | 出願番号 |
| `AMAPP` | 出願人 |

#### 範囲検索
`AD:2011-01-01~2011-03-01` のようにチルダで範囲
不等号: `>` / `<` / `=`

#### 分類階層指定
- `\*` 前方一致
- `.` 下位分類

#### セマンティック検索
- `SE:"AN:JP20090253196"` 類似文献検索
- `SEC:` 明細書除外での類似検索

#### 行参照 / コメント
- `L:{行番号}` 前行の集合を参照（`L:1 OR L:2`）
- `//` コメント

#### 例
```
KW:太陽電池 AND KW:パネル
KW:*N5"太陽 電池"
AB:*ONP5"(太陽 ソーラー) (電池 パネル)"
L:1 OR L:2
IPC:G06F*
AD:2020-01-01~2025-12-31
```

#### 識別用の構文特徴
- 行頭に `KW:` `CL:` `IPC:` 等の **コード + コロン**
- 近傍演算子が `*N{n}` / `*ONP{n}` 形式（アスタリスク + N/ONP）
- `L:1` 行参照

**Sources**:
- [Patentfield コマンド検索（公式ヘルプ）](https://en.patentfield.com/help/functions-command-search)
- Patentfield サポートセンター: 近傍検索, 詳細検索画面

### 2-4. Shareresearch（日本商用）

**提供元**: 株式会社日立社会情報サービス（SRPARTNER は日立システムズ提供の類似サービス、同じ検索エンジンベース）
**位置づけ**: **国内文字数・国外単語数** の距離文化を持つ日本商用 DB。論理式と画面方式の併用型

#### 演算子
| 種別 | 記法 |
|---|---|
| 画面方式 | `一致検索(OR)` / `一致検索(AND)` |
| 式中 | `*` / `+` ※公開資料では**意味対応が明示されていない**（`*`=AND、`+`=OR と読むのが自然だが実機ヘルプでの最終確認推奨） |
| 集合演算 | `#` + 履歴番号（`S1*S2` 等） |

#### 近傍演算子
| 記法 | 意味 |
|---|---|
| `adj{n}` | **語順固定** |
| `near{n}` | **語順不問** |

**距離単位**: **国内 = 文字数、国外 = 単語数**
**省略時距離**: 国内 1、国外 0
**制限**: 半角小文字のみ有効、前後スペース不可

#### フレーズ / 引用
- `"..."` でフレーズ、スペースを含むキーワードを囲む
- キーワード側の論理演算子が **画面方式指定より優先**

#### ワイルドカード
- 英文前方一致: `absorb*`
- 日本語はデフォルト**部分一致**
- 本文系特殊記号: `?`、`|`、`^`

#### フィールド指定
画面で選択：
- IPC / IPC8 / 更新 IPC / 更新 IPC8
- FI
- F ターム（`$` で完全一致: `4C083AC43$`）
- 出願人／権利者（公開データ）
- 現出願人／権利者（最新データ、`|` で単独／共同の区別）
- 公開日 / 優先日 / 出願日
- 審査・権利状況（`0` = 権利消滅、`1` = 生存）

#### 制限
- 1 検索式欄 最大 30 キーワード
- 括弧不整合、演算子省略、演算子連続はエラー
- 括弧の中に演算子が無い場合は文字列扱い

#### 例
```
(Ａ+Ｂ)adj3Ｃ
(Ａ+Ｂ)nearＣ
"plasma display"
absorb*
S1*S2
4C083AC43$
```

#### 識別用の構文特徴
- `adj{n}` / `near{n}` 近傍演算子
- F ターム末尾 `$` 完全一致
- `S1*S2` 集合演算

**Sources**:
- [Shareresearch オンラインマニュアル 3 章](https://www.srpv2.com/SR-C/Help/guidetour/guidetour_Chap3.pdf)
- [Shareresearch 演算子の注意事項](https://www.srpv2.com/SR-C/Help/kanren/k02030101.html)
- [SRPARTNER 検索項目 FAQ（日立システムズ）](https://www.hitachi-systems.com/ind/srpartner/faq/faq06/)

### 2-6. PatentSQUARE（日本商用）

**提供元**: パナソニック ソリューションテクノロジー株式会社
**位置づけ**: 1992 年から提供の日本商用 DB。**スペース区切りが暗黙の OR** という独特の仕様

#### 演算子
| 種別 | 半角 | 全角 |
|---|---|---|
| AND | `*` | `＊` |
| OR | `+` | `＋` |
| NOT | `#` | `＃` |

**⚠️ 特殊仕様**: **複数ワードをスペース区切りで入力すると自動的に OR（＋）に変換される**

#### 括弧
- `( )` 使用可

#### 近傍演算子
公式 FAQ で「近傍検索機能を標準搭載」と明記。詳細構文は公開 FAQ だけでは追いにくい。補助入力画面では文字数指定 + NOT チェックボックスで設定可能。**実機ヘルプでの最終確認推奨**。

#### ワイルドカード
- `?キーワード?` **中間一致**

#### フィールド指定
画面で選択（プロフェッショナル検索）：
- 全文
- 出願人・権利者名
- 発明者名
- IPC / FI / F ターム
- 公開・登録番号
- 経過情報
- 引用情報
- Family

#### 集合番号
検索履歴番号で集合演算: `S001*S002*S003`（AND）、`S001+S002`（OR）

#### 制限
- 1 行最大 2000 バイト
- 最大 999 行

#### 例
```

?キーワード?
S001*S002*S003
太陽 電池                // 自動的に「太陽 OR 電池」として解釈（注意）
```

#### 識別用の構文特徴
- 全角演算子 `＊` `＋` `＃` が出現

## 3. 共通原則・注意点

### 3-1. 演算子の記号対応（DB 別一覧）

| DB | AND | OR | NOT |
|---|---|---|---|
| J-PlatPat | `*` | `+` | `-` |
| JP-NET | `&` / `AND` | `+` / `OR` | `!` / `NOT` |
| Patentfield | `and` / `+` / 省略 | `OR` | `not` / `-` |
| Shareresearch | `*`（推定）/ `一致(AND)` | `+`（推定）/ `一致(OR)` | 明示なし |
| BizCruncher | `*` / `&` | `+` | `!` |
| PatentSQUARE | `*` / `＊` | `+` / `＋` / **スペース自動 OR** | `#` / `＃` |
| PatSnap | `AND` | `OR` | `NOT` |
| Espacenet | `AND` | `OR` | `NOT` |
| PATENTSCOPE | `AND` | `OR` | `ANDNOT` |

### 3-2. 近傍演算子の距離単位

| DB | 日本語 | 英語 |
|---|---|---|
| J-PlatPat | 文字数 | 単語数 |
| JP-NET | 文字数 | 単語数 |
| Patentfield | 文字数 | 単語数 |
| Shareresearch | **文字数（国内）** | **単語数（国外）** |
| BizCruncher | 文字数（国内） | 単語数（国外）※推定 |
| PatentSQUARE | 文字数（推定） | 単語数（推定） |
| PatSnap | -- | 単語数 |

**共通パターン**: 日本系 DB は日本語対応のため文字数単位、英語中心 DB（PatSnap / Espacenet 等）は単語数単位

### 3-3. DB 固有の「罠」

- **PatentSQUARE**: スペース区切りが **暗黙の OR**（他の DB では AND だと思い込むと誤読する）
- **Patentfield**: 近傍式の内部は入れ子・AND/OR/NOT 非サポート（複雑な式を近傍に組み込めない）
- **Shareresearch**: `*`/`+` の意味対応が公開資料で断定できない（実機ヘルプ推奨）
- **J-PlatPat vs JP-NET**: 名前が似ているが別物。J-PlatPat は INPIT（特許庁系）無償、JP-NET は JPDS（日本パテントデータサービス株式会社）商用
- **PATENTSCOPE**: NOT ではなく `ANDNOT`（他 DB から移植時の罠）
- **PatSnap**: `$W` の `$` を見落とすと近傍演算子が読めない

## 4. B: 意図 ↔ 論理 乖離検出チェックリスト

Phase A の **STOP-GATE B** で、エージェントは `query_intent`（自然言語の設計意図）と `query_logic`（実装された検索式）を対比し、以下のチェックリストで乖離を検出する。

### 4-1. 乖離の 3 段階分類

| 重大度 | 例 | 処理 |
|---|---|---|
| 🔴 **Critical（矛盾）** | 意図に「食品用途除外」とあるが検索式に食品 IPC 除外がない | STOP-GATE で必ず提示、ユーザーが「修正 / このまま進める / 無視」を選択 |
| 🟡 **Warning（過剰/不足）** | 意図に記載ない IPC が検索式で除外されている（過剰絞り込み） | まとめて提示、同様にユーザー選択 |
| 🔵 **Info（解釈の幅）** | 意図の「構造材料用途」が C08L を含むか曖昧 | 「本分析の範囲と限界」に注記する旨を伝えるのみ |

**重要**: **Critical 検出時でも進行可能**（ユーザー判断を尊重）。ただしエージェントは「検索式をこう修正すると意図に沿う」という **具体的な改善提案** を添えること。

### 4-2. 整合性チェック 8 項目

各項目について、`query_intent` 側と `query_logic` 側を対比する：

| # | 意図側のキーワード | 論理側で検出すべきもの | 乖離の例 |
|---|---|---|---|
| 1 | **技術領域の指定**（例: 「CNF 構造材料」）| IPC / FI / CPC / F ターム / キーワード条件 | 意図に「CNF」とあるが分類・キーワードに CNF 相当が見当たらない → Critical |
| 2 | **用途・応用分野の指定**（例: 「構造材料として」）| 関連 IPC / キーワード条件 | 意図に「食品除外」とあるが A23 系 IPC の除外がない → Critical |
| 3 | **対象期間**（例: 「直近 5 年」「2020 年以降」）| AD / PD / PRD 等の日付条件 | 意図に「直近動向」とあるが期間指定がない → Warning |
| 4 | **地域/国の指定**（例: 「日本特許中心」）| 国指定 / 公報種別 / データベース名 | 意図に「グローバル比較」とあるがデータベースが日本のみ → Critical |
| 5 | **対象出願人の絞り込み**（例: 「主要 5 社のみ」）| PA / AS / assignee 条件 | 意図に「3 社限定」とあるが検索式に出願人絞り込みがない → Warning |
| 6 | **除外条件**（例: 「特定用途を除外」）| NOT 条件 | 意図に「除外」の言及があるが検索式に NOT がない → Critical |
| 7 | **公報種別**（例: 「登録特許のみ」「公開公報も含む」）| 公報種別コード / 審査状態 | 意図に「登録済のみ」とあるが検索式に状態指定がない → Warning |
| 8 | **分類の階層深度**（例: 「サブクラスまで絞る」）| IPC 階層指定（ドット / 範囲 / 筆頭 vs 全体） | 意図に「メイングループまで」とあるが筆頭 IPC のみ指定 → Info |

### 4-3. 乖離提示時の推奨アドバイス形式

エージェントがユーザーに提示する乖離報告は、以下のテンプレートに沿う：

```
🔴 乖離 #1 [Critical]: 食品用途の除外が不整合

■ 意図側 (query_intent より抽出):
  「本母集団は CNF のうち食品・化粧品用途を除外し、構造材料用途に焦点を絞る」

■ 論理側 (query_logic より抽出):
  C08L*/FI * C08K*/FI * B29*/FI
  (食品用途を除外する NOT 条件なし)

■ 乖離の内容:
  意図は食品用途除外を明示しているが、検索式に NOT 条件が存在しない。
  結果として、食品用途の CNF 特許が母集団に混入している可能性がある。

■ 改善提案:
  検索式の末尾に以下を追加することを推奨：
  * NOT (A23*/FI + A21*/FI)  // 食品系 IPC の除外
  * NOT (A61K8*/FI + A61Q*/FI)  // 化粧品系 IPC の除外

■ 選択肢:
  [A] 検索式を修正して APOLLO で再抽出する（推奨）
  [B] このまま進め、「本分析の範囲と限界」章で乖離を明記する
  [C] 乖離は想定内として無視する
```

## 7. 既知の落とし穴

### 7-1. DB 名の誤認

- **J-PlatPat と JP-NET を混同しないこと**: 名前が似ているが別物。J-PlatPat は INPIT（特許庁系）の無償サービス、JP-NET は JPDS（日本パテントデータサービス株式会社）の商用サービス
- **IPC と CPC の違い**: 日本系 DB は IPC 中心、欧米系 DB（Espacenet / USPTO）は CPC を併用
- **F ターム vs FI**: F タームは日本特許庁独自の技術分類、FI は日本特許庁の拡張 IPC

### 7-2. 前方一致と完全一致

- ワイルドカード `*` の位置（前・中・後）で挙動が変わる
- Shareresearch では F ターム末尾の `$` が完全一致を示す（他 DB にはない独自記法）
- PatSnap の `#` は NOT ではなく「0 または 1 文字」のワイルドカード（混同注意）

### 7-3. 英語 DB での日本語検索

- PatSnap / Espacenet / Google Patents などグローバル DB で日本語特許を検索する場合、英訳を使うのが標準
- 日本系 DB（J-PlatPat / JP-NET / Patentfield 等）は日本語ネイティブ対応

### 7-4. 地域偏り

- `database_name` が日本限定 DB（J-PlatPat / JP-NET / Patentfield 等）の場合、検索式に国指定がなくても **結果は日本特許に限定**される
- 意図に「グローバル比較」とあればこれは Critical 乖離として B で検出すべき

### 7-5. 公開情報の限界

- 本ガイドは 2026-04-19 時点の公開情報ベース。各 DB のログイン後マニュアル・契約者向け資料までは参照していない
- 特に **BizCruncher / Shareresearch / PatentSQUARE** は公開情報で確定できない部分がある。実機ヘルプでの最終確認を推奨
- 本ガイドに記載のない DB（PatSeer / Minesoft など）を検出した場合は、執筆者が自然言語で推測しつつ、ユーザーに「この DB の読解で合っているか」を確認する

# 可視化読解ガイド

各モジュールの可視化（UMAP散布図、共起ネットワーク、4象限マップ等）から何を「見て」何を「考える」かのステップバイステップ手順。4層分析モデル（→ common_framework.md）を各可視化に適用する方法を示す。

> **選択的読み込み**: 本ファイルは6種類の可視化読解ガイドを含む。**全セクションを一度に読む必要はない。** 分析対象の可視化に対応するセクション番号のみを読むこと。
> - Saturn V / EAGLE分析時 → 共通原則 + セクション1のみ
> - Explorer分析時 → 共通原則 + セクション2のみ
> - MEGA分析時 → 共通原則 + セクション3のみ
> - CREW分析時 → 共通原則 + セクション4のみ
> - ATLAS / NEBULA分析時 → 共通原則 + セクション5のみ
> - ワードクラウド分析時 → 共通原則 + セクション6のみ

---

## 読み方の共通原則

### 3段階ズーム

1. **マクロ（全体俯瞰）**: 全体の形状・分布パターン・バランス
2. **メソ（構造把握）**: グループ間の関係・ブリッジ・ギャップ
3. **ミクロ（個別特徴）**: 異常値・外れ値・注目すべき個体

### 4つの空間的視点

| 視点 | 着眼点 | 意味するもの |
|------|--------|------------|
| 近さ | 近接する要素同士 | 技術的類似性、協業関係、関連キーワード |
| 遠さ | 離れた要素同士 | 異なる技術領域、断絶、潜在的融合機会 |
| 密集 | 要素が集中するエリア | 活発な研究領域、レッドオーシャン、確立された技術 |
| 孤立 | 他から離れた単独要素 | ニッチ技術、先駆的研究、ノイズ |

## 1. UMAP散布図 (Saturn V / EAGLE)

**データソース**: `saturnv_clusters.json` の `spatial_context` + `clusters` / `eagle_clusters.json`

### 5つの読解ステップ

**Step 1: 全体形状の把握**
- 散布図全体はどんな形をしているか（円形/楕円/帯状/複数の島）
- 点の分布に偏りはあるか（一箇所に集中/均等に分散）
- → 技術領域全体の構造を把握する

**Step 2: 空間グルーピングの確認**
- `spatial_context.spatial_groups` を確認: どのクラスタが空間的に近いか
- 近接するクラスタ群 = 上位の技術カテゴリ（例: 「半導体プロセス群」「材料群」）
- クラスタ間の距離が近い → 技術融合の可能性がある
- → 技術領域間の関係性を読み解く

**Step 3: 孤立クラスタの特定**
- `spatial_context.isolated_clusters` を確認
- 孤立 = 他の技術領域と異質な独自技術
- 孤立クラスタのサイズが大きい → 確立された独自領域
- 孤立クラスタのサイズが小さい → ニッチまたはノイズ
- → 独自技術の発見・評価

**Step 4: 境界領域の注目**
- `spatial_context.bridge_clusters` を確認
- 複数の空間グループにまたがるクラスタ = 技術横断的な領域
- ブリッジクラスタは技術融合のハブになり得る
- → イノベーション機会の特定

**Step 5: ノイズ点の解釈**
- cluster = -1 の点の分布
- ノイズが多い = 分類しきれない多様な技術が存在
- ノイズが特定エリアに集中 = 新興技術の萌芽の可能性
- → 潜在的な新規クラスタの候補
- **深掘り**: `saturnv_clusters.json` の `noise_patents` 配列（上限200件）にノイズ特許の座標・タイトル・出願人・年が含まれる。`analysis/noise_analysis.md` の5手法（ノイズ率解釈・空間分布・時系列・出願人分析・新興領域抽出）を適用し、ノイズの中から新興技術の萌芽を発見する

### 4層モデル適用例

> **事実**: Saturn V TELESCOPE分析で12クラスタが検出された。spatial_contextによると、クラスタ1・3・7は空間的に近接してグループを形成し、クラスタ9は最も孤立している（最近接クラスタとの距離が平均の2.3倍）。
>
> **解釈**: クラスタ1（半導体プロセス, 152件）、3（エッチング技術, 87件）、7（薄膜形成, 64件）は同一の上位技術カテゴリ「半導体製造プロセス」に属すると考えられる。クラスタ9（量子コンピューティング, 23件）は独自の技術体系を持ち、従来の半導体技術とは異なるアプローチを採用している。
>
> **So What?**: 半導体製造プロセス群（クラスタ1+3+7, 計303件）は当該特許群の中核をなす。一方、量子コンピューティング（クラスタ9）は孤立しているが、将来的にクラスタ1との融合（量子半導体）が起こり得る。
>
> **提言**: 半導体製造プロセス群の3クラスタの境界領域（例: エッチング×薄膜の複合プロセス）に出願空白があれば、ミクロ分析によると（patents.csvで確認）、優先的な出願機会と捉えるべきである。

> **レポートへの反映**: `capcom_schema/exemplars/saturnv_exemplar.md` を読み、同等の深さ・構造・ナラティブ品質で記述する。

## 2. 共起ネットワーク (Explorer)

**データソース**: `explorer_global_network.json` の `nodes` / `edges` / `communities`

### 5つの読解ステップ

**Step 1: コミュニティ構造の把握**
- `communities` の数とサイズバランス
- 大きなコミュニティ = 確立された技術テーマ
- 多数の小コミュニティ = 技術の多様性・断片化
- → 技術領域のセグメンテーション

**Step 2: ハブノードの特定**
- `nodes` の `degree`（次数）上位を確認
- ハブ = 多くの技術キーワードと共起する汎用的概念
- ハブが1つ → 中心集約型の技術体系
- ハブが複数 → 多極型の技術構造
- → 技術の中核概念を特定

**Step 3: ブリッジエッジの発見**
- 異なるコミュニティ間を繋ぐエッジ
- ブリッジ = 技術領域間の接点・融合ポイント
- ブリッジのweight（共起頻度）が高い → 確立された技術融合
- ブリッジのweightが低い → 萌芽的な融合
- → 技術融合の現状と機会

**Step 4: エッジの太さ（weight）の分布**
- weight上位のエッジ = 強固な技術的関連
- 全体的にweightが均一 → 技術キーワード間の関連が均等
- 少数のエッジにweightが集中 → コアとなる技術概念ペアが存在
- → 技術の結びつきの強弱

**Step 5: 周辺・孤立ノードの評価**
- degree = 1-2 の周辺ノード
- 周辺だが新しいキーワード → 新興技術の予兆
- 孤立ノード = 他と共起しない独自概念
- → 新興技術・ニッチ分野の発見

### 構造パターン分類

| パターン | 特徴 | 解釈 |
|---------|------|------|
| スター型 | 1つのハブに多数のノードが接続 | 1つの中核技術を中心とした体系 |
| クリーク型 | 複数ノードが相互に密接続 | 成熟した技術クラスタ（研究が収束） |
| チェーン型 | ノードが直列に接続 | 技術の段階的発展・プロセスチェーン |
| 孤立島型 | 複数の切断されたサブグラフ | 技術の断絶・独立した研究領域 |

### 4層モデル適用例

> **事実**: 共起ネットワークは5つのコミュニティに分かれ、最大コミュニティ（15ノード）の中心は「リチウムイオン」（degree 12）。「固体電解質」と「正極材料」を繋ぐブリッジエッジ（weight 23）が存在する。
>
> **解釈**: 「リチウムイオン」が技術体系の中核をなし、周囲の「正極」「負極」「電解質」「セパレータ」が電池部材の技術マップを構成している。「固体電解質」と「正極材料」のブリッジは全固体電池の開発動向を反映している。
>
> **So What?**: 全固体電池は既存のリチウムイオン電池技術とブリッジで繋がりつつも、独自のコミュニティを形成しつつある。このブリッジの強化（共起頻度の増加）が今後の技術トレンドを示す先行指標となる。
>
> **提言**: 「固体電解質—正極材料」ブリッジエッジのweight推移をExplorer Trendで確認し、急上昇していれば全固体電池への重点投資を検討すべきである。

> **レポートへの反映**: `capcom_schema/exemplars/explorer_exemplar.md` を読み、同等の深さ・構造・ナラティブ品質で記述する。

## 3. 4象限マップ (MEGA PULSE)

**データソース**: `mega_momentum.json` の `quadrant_summary` / `entities`

### 5つの読解ステップ

**Step 1: 4象限の分布バランス**
- 各象限のエンティティ数と件数割合
- QI（成長×活発）に集中 → 全体的に成長産業
- QIII（衰退×低活動）に集中 → 成熟・衰退産業
- 均等分布 → 産業の転換期
- → 技術領域全体のライフサイクル段階

**Step 2: 密度の高いエリア**
- 同一象限内でのエンティティ集中
- CAGR・Activity軸上の値が近接 → 類似した動態の競合
- → 競争激化エリアの特定

**Step 3: 新興プレイヤーの発見（QII）**
- QII（高成長・低活動量）のエンティティ
- 少数の出願で高い成長率 → 新興参入者または新技術
- CAGR極端に高い（>50%/年）→ 基数が小さい可能性に注意
- → 将来の脅威・機会の特定

**Step 4: 象限境界のエンティティ**
- CAGR ≒ 0 付近のエンティティ → 成長停滞の転換点
- Activity軸の中央付近 → 分析期間によって象限が変わり得る
- → 不安定な状態にあるプレイヤーの特定

**Step 5: バブルサイズ（件数）の比較**
- 大バブル = 出願件数が多い主要プレイヤー
- 小バブルだがQI → 少数精鋭の成長プレイヤー
- 大バブルだがQIII → かつての主要プレイヤーの衰退
- → プレイヤーの規模と動態の関係

### 4層モデル適用例

> **事実**: MEGA PULSEの4象限分析で、A社はQI（CAGR +18.5%, Activity 450件）、B社はQIV（CAGR -3.2%, Activity 380件）、新興C社はQII（CAGR +45.2%, Activity 28件）に位置する。
>
> **解釈**: A社は成長投資を継続する攻撃的ポジション。B社は大量の出願を維持しながらも成長が停滞し、防衛的出願フェーズにある。C社は少数だが急成長しており、特定ニッチ領域への集中投資が推察される。
>
> **So What?**: B社の停滞は市場シェア獲得のチャンスだが、既存出願件数（380件）による防壁は依然として高い。C社の急成長領域がA社・B社の主力領域と重なる場合、将来的な競合激化が予想される。
>
> **提言**: ミクロ分析によると、C社のQII出願28件はクラスタ5（量子ドット材料）に集中している。A社は同クラスタに7件しかなく、C社との技術ギャップが拡大する前に、共同研究または自社出願の強化（目標: 年間10件）を検討すべきである。

> **レポートへの反映**: `capcom_schema/exemplars/` のお手本ファイルを参考に、同等の深さ・構造・ナラティブ品質で記述する。

## 4. 人的ネットワーク (CREW)

**データソース**: `crew_network.json` の `nodes` / `edges` / `communities` / `metadata`

### 5つの読解ステップ

**Step 1: ネットワーク密度の把握**
- `metadata.density` を確認
- 高密度（>0.3）→ 緊密な協業エコシステム
- 低密度（<0.1）→ 分散した個別研究が多い
- → 業界の協業文化の評価

**Step 2: 媒介中心性（betweenness）上位者**
- betweenness上位のノード = ネットワーク全体の情報ブローカー
- 複数コミュニティに跨る人物 → 技術横断的な知見を持つキーパーソン
- 1社内にブローカーが集中 → 社内統合力が高い
- → キーパーソンの特定

**Step 3: コミュニティ構造**
- コミュニティ = 共同研究グループ / 企業内チーム / アライアンス
- 大きなコミュニティ = 活発な研究グループ
- コミュニティ間の接続 → 組織間連携
- → 研究体制・アライアンスの可視化

**Step 4: Rising Stars（新興研究者）**
- degree（共著者数）は少ないが、直近の出願に集中して登場する人物
- 新しい研究チームの中心メンバーの可能性
- → 新興研究勢力の早期検知

**Step 5: 孤立ノードの解釈**
- degree = 1-2 の周辺ノード
- 単独発明者 → 個人研究者または外部顧問
- 大企業の孤立研究者 → 独立した研究テーマの担当者
- → 特殊な研究リソースの発見

### 4層モデル適用例

> **事実**: CREWネットワーク（元データ: 450ノード・1,200エッジ、要約版: 50ノード・200エッジ）で、山田氏（A社）のbetweenness 0.42が最高値。3つのコミュニティ（A社チーム・B社チーム・大学連合）を繋いでいる。
>
> **解釈**: 山田氏はA社に所属しながら、B社との共同研究および大学との産学連携の両方でブリッジ役を果たしている。同氏の退職・異動はネットワーク分断のリスクを伴う。
>
> **So What?**: A社のオープンイノベーション戦略は山田氏個人に大きく依存しており、組織的なリスクがある。
>
> **提言**: 山田氏の共同研究パートナーを組織的に取り込み、複数人でブリッジ役を担う体制への移行を推奨する。ミクロ分析によると、山田氏の共著特許18件のうち12件が固体電解質関連であり、この領域の後継者育成が最優先である。

> **レポートへの反映**: `capcom_schema/exemplars/` のお手本ファイルを参考に、同等の深さ・構造・ナラティブ品質で記述する。

## 5. トレンド・ライフサイクル (ATLAS / NEBULA)

**データソース**: `atlas_statistics.json` の `time_series` / `lifecycle` / `nebula_hype_cycle.json`

### 5つの読解ステップ

**Step 1: 全体傾向の把握**
- 年別出願件数の推移
- 右肩上がり → 成長産業
- ベルカーブ → 成熟→衰退
- 波状 → 技術サイクルの繰り返し
- → 産業全体の成長段階

**Step 2: 変曲点の特定**
- 傾きが大きく変化した年
- 急増 → 規制変更・技術ブレイクスルー・新規参入者
- 急減 → 市場撤退・技術陳腐化・代替技術の出現
- → トレンド転換のトリガー特定

**Step 3: ピーク年の確認**
- 出願件数最大の年
- ピーク後に減少 → 技術の成熟・飽和
- ピーク後に横ばい → 安定した市場需要
- ピークが直近 → まだ成長中の可能性
- → 市場タイミングの評価

**Step 4: 直近年の注意**
- 最新1-2年の出願数は、公開遅延（出願から18ヶ月）により実態より少なく見える
- 直近の減少が見かけ上のものか実質的なものか判断する
- → データの信頼性の確保

**Step 5: 特許-NPLタイムラグ (NEBULA)**
- 論文のピークと特許のピークの時間差
- 論文が先行 → 基礎研究から応用への展開段階
- 同時 → 研究と事業化が並行
- 特許が先行 → 実用化主導の領域
- → 技術移転のスピード評価

### 4層モデル適用例

> **事実**: ATLAS時系列分析で、出願件数は2018年の120件から2022年の310件へ増加（CAGR +26.8%/年）。2023年は公開遅延を考慮しても推定280件以上。ライフサイクル分析ではIPCクラスH01L21が「成長期」に分類。
>
> **解釈**: 半導体製造プロセス関連の出願は4年間で2.6倍に増加しており、業界全体で活発な技術開発が進行中。公開遅延を考慮しても2023年の実質出願数は前年と同水準以上と推定される。
>
> **So What?**: 成長率 +26.8%/年は持続可能な水準を超えている可能性があり、今後2-3年で成長鈍化（成熟期への移行）が起こり得る。市場参入のウィンドウは限られている。
>
> **提言**: NEBULAの論文データと照合し、論文出版ピークが既に過ぎている場合は、基礎研究段階は終了しており出願の重点を実装・製造プロセスに移すべきである。

> **レポートへの反映**: `capcom_schema/exemplars/` のお手本ファイルを参考に、同等の深さ・構造・ナラティブ品質で記述する。

## 6. ワードクラウド (複数モジュール)

**データソース**: `data/*_wordcloud.json`（Saturn V PROBE / Explorer / MEGA / EAGLE / CREW で生成）

### 5つの読解ステップ

**Step 1: 高頻出語の把握**
- `word_frequencies` の上位10-15語を抽出する
- 最頻出語 = その技術領域の中核概念
- 上位語に汎用的な語が多い → ストップワードの追加を検討
- → 技術領域の中核テーマを特定

**Step 2: 頻出語の階層構造**
- 上位語（100回以上）= 確立された主要概念
- 中位語（30-100回）= 技術のサブカテゴリ
- 下位語（10-30回）= ニッチ・新興トピック
- → 技術の成熟度を3層で評価

**Step 3: 複数モジュール間の比較**
- 同じ単語が複数のワードクラウドに出現 → 横断的な技術概念
- 特定モジュールでのみ高頻出 → そのモジュール固有の視点
- 例: Saturn V PROBEとExplorerの共通語 = SBERTクラスタと共起ネットワークの両方で重要なキーワード
- → モジュール間の一貫性・差異を評価

**Step 4: 技術用語 vs 機能用語の分離**
- 技術用語（材料名、装置名、プロセス名）→ 具体的な技術内容
- 機能用語（制御、管理、最適化、効率）→ 技術の方向性・課題
- 技術用語が優勢 → 基礎研究段階の特許群
- 機能用語が優勢 → 実装・応用段階の特許群
- → 技術のライフサイクル段階を推定

**Step 5: 不在の語（ネガティブスペース）**
- 期待されるが出現しない語は何か
- 例: 水素貯蔵技術で「安全」が低頻度 → 安全技術は別領域で扱われている / 暗黙知化している
- 競合他社名や特定技術が不在 → その領域の空白（ホワイトスペース）の候補
- → 潜在的な機会・盲点の発見

### 4層モデル適用例

> **事実**: Saturn V PROBEのクラスタ4（161件）のワードクラウドで、「水電解」（出現87回）、「太陽光」（52回）、「変動」（41回）が上位を占める。一方、「蓄電池」（8回）、「系統」（6回）は下位にとどまる。
>
> **解釈**: このクラスタは再生可能エネルギーと水電解の連携技術が中核であり、太陽光の出力変動対応が主要な技術課題である。蓄電池との組合せは検討されているが主流ではない。
>
> **So What?**: P2G（Power-to-Gas）技術は蓄電池との競合ではなく補完として位置づけられており、大規模・長時間貯蔵の需要に対応する技術として差別化されている。
>
> **提言**: Explorerのワードクラウドとの比較で「変動」が共起ネットワークでも成長キーワードであることを確認し、変動吸収制御技術への投資を優先すべきである。

> **レポートへの反映**: 各deep_diveのミクロ分析セクション等で、ワードクラウドの特徴語を定量的根拠として活用する。

## クラスタ動態マップの読み方

### チャートタイプ
X=累積件数（クラスタの規模）、Y=CAGR（直近N年の成長率）のバブルプロット。
バブルサイズはクラスタのシェア（全体に占める割合）。

### 5ステップ読解法

**Step 1: 象限分布のバランス**
4象限にクラスタがどう分布しているかを確認。成長リーダーに偏っていれば技術全体が活況。ニッチ/衰退に偏っていれば成熟期。

**Step 2: 成長リーダー（右上）の同定**
大規模×高成長のクラスタが技術の主戦場。ここに位置するクラスタのラベルが、当該分野の中核テーマ。

**Step 3: 新興クラスタ（左上）の注目**
小規模だが急成長。萌芽技術の候補。Saturn Vのノイズ分析と照合し、ノイズからクラスタに成長した技術を特定。

**Step 4: 成熟クラスタ（右下）の評価**
大規模だが成長鈍化。差別化が難しくなっている領域。改良発明が中心か、破壊的イノベーションの余地があるかを検討。

**Step 5: ニッチ/衰退（左下）の判断**
撤退候補か、逆にニッチ戦略で独占できる領域か。出願人分析と合わせて判断。

### Saturn V vs EAGLE vs NEBULA の動態マップ比較
同一データに対して異なるクラスタリング（自動 vs 手動 vs 学術）の動態を比較することで:
- 自動クラスタリングでは見えない成長パターンの発見
- 研究（学術）→ 開発（特許）のタイムラグの定量化
- 分析者の仮説（EAGLE手動クラスタ）の定量的検証

# 共通分析フレームワーク

レポート執筆開始時に最初に読むこと。全ての分析ファイル（map_reading.md / cross_module.md / patent_citation.md）の基盤となる。

---

## 1. 4層分析モデル

全ての分析は以下の4層を必ず経由すること。Layer 1だけで終わるレポートは「事実の列挙」でしかない。

| Layer | 名称 | 内容 | 文章中のマーカー例 |
|-------|------|------|-----------------|
| Layer 1 | 事実 | データから直接読み取れる客観的数値・パターン | 「〜である」「〜件」「〜%」 |
| Layer 2 | 解釈 | 技術的・市場的に何を意味するか | 「これは〜を示唆している」「〜と解釈できる」 |
| Layer 3 | So What? | 戦略的示唆・ビジネスインパクト | 「したがって〜」「戦略的には〜」 |
| Layer 4 | 提言 | 具体的アクション・推奨事項 | 「〜を推奨する」「〜を検討すべきである」 |

### NG例（Layer 1止まり）

> クラスタ3は152件の特許を含み、主要出願人はA社（45件）とB社（32件）である。IPCはH01L21が最多で、2020年以降増加傾向にある。

### OK例（4層すべて）

> クラスタ3は152件の特許を含み、A社（45件, 29.6%）とB社（32件, 21.1%）で過半数を占める（**事実**）。両社ともH01L21（半導体製造プロセス）に集中しており、次世代製造技術の覇権争いが進行中と解釈できる（**解釈**）。MEGA PULSEで同クラスタはQuadrant I（高成長・高活動）に位置し、今後3-5年の出願増加が見込まれるため、この領域でのポジション確保が競争優位の鍵となる（**So What?**）。自社のH01L21関連出願を年間10件→20件に倍増させ、特にA社が手薄なウェットエッチング工程（ミクロ分析によると、A社の45件中わずか3件）への重点投資を推奨する（**提言**）。

### 4層モデルの実践チェック

各deep_diveセクションで、以下の自己チェックを行うこと:

1. **L1（事実）が数値付きか**: 「多い」「少ない」等の形容詞は禁止。必ず件数・割合・CAGR等の数値を含める
2. **L2（解釈）に「なぜ」があるか**: 数値変動の因果（政策変更、技術成熟、企業戦略転換等）を論じているか
3. **L3（洞察）に「にもかかわらず」「一見〜だが」があるか**: 単純でない、複数事実を統合した知見を含んでいるか
4. **L4（提言）に企業名・数値目標があるか**: 「検討すべき」だけでなく、誰が・何を・いつまでにを含んでいるか

### 🔍 各 Layer のスコープ明示ルール（絶対遵守）

4 層モデルを使うときに最も頻発するバグが、**「本母集団内の観察」をいつの間にか「業界全体の傾向」として書いてしまう**こと。各 Layer ごとにスコープを以下の通り限定する（詳細は `terminology.md` §6）:

| Layer | スコープ | 書き方ルール |
|---|---|---|
| L1（事実） | **本母集団のみ** | 必ず「本母集団では〜」「本分析の特許群では〜」等の限定修飾を置く。「業界では A 社が 1 位」は NG、「本母集団では A 社が最多出願（152 件、23.4%）」は OK |
| L2（解釈） | **本母集団の解釈** | 同様に母集団限定を維持。「なぜ本母集団でこのパターンが見られるか」を論じる |
| L3（洞察） | 母集団限定が原則、一般化は条件付き | 業界全体への一般化を書くなら、Web 調査の外部裏付けを `#footnote` で必ず付す。裏付けなしに「業界はこの方向」と断定するのは品質不合格 |
| L4（提言） | 提言の対象と根拠を明示 | 「本母集団で観察された動向に基づき、A 社は〜」のように、提言の導出根拠が母集団観察か外部一般化かを明示する |

**NG 例（Layer 違反、スコープ無限化）**:

> 業界では半導体製造プロセスが EUV リソグラフィに集約されつつある（L1）。これは A 社の先行投資が効いており（L2）、今後 5 年で業界の標準技術として確立するだろう（L3）。各社は追随投資を検討すべきである（L4）。

この例は L1 が「業界では」と無限化、L3 が外部裏付けなしに「業界の標準として確立する」と断定、L4 が「各社」と主語を広げているため、3 つの Layer で誤読を誘発している。

**OK 例（Layer 別のスコープ維持）**:

> 本母集団では、EUV リソグラフィ関連の出願がクラスタ 3 に集中し、152 件（全体の 23.4%）を占める（L1: 母集団限定の事実）。これは A 社（45 件）と B 社（32 件）で過半数を占めており、両社が次世代半導体製造の主導権を争っていると解釈できる（L2: 母集団内の因果）。外部市場調査では EUV 設備投資が世界全体で年 30% 増加しており#footnote[SEMI "World Fab Forecast" (URL), 取得日: 2026-04-14]、本母集団の出願集中は業界全体の設備投資動向と整合的である（L3: 外部裏付け付きの一般化）。従って、本母集団に出願の少ない C 社は、EUV 領域への R&D 投資を 2025 年度から年 20 億円規模で立ち上げることを推奨する（L4: 母集団観察と外部データに基づく提言）。

### 🎯 設計意図（query_intent）を 4 層モデルで内在化させる

Phase A STOP-GATE で確定した `query_intent` と `sub_questions`（`reports/_phase_a_decisions.json`）は、**4 層モデルの各層で継続的に意識する**こと。特に Layer 3（洞察）と Layer 4（提言）で意図と乖離した結論を書くと、レポート全体の一貫性が失われる。

| Layer | 設計意図の扱い |
|---|---|
| **L1（事実）** | 意図に直接関係する事実を優先的に取り上げる。ただし意図から外れる事実も観察された場合は隠さず記述（後段で「分析過程で確認された追加的事項」章にまとめる） |
| **L2（解釈）** | 「本分析の視座では〜」等の修飾を Layer 1 → Layer 2 の移行で添えると、意図との接続が自然になる |
| **L3（洞察）** | 意図に対して何を示唆するかを明示する。意図と無関係な洞察を書かない |
| **L4（提言）** | **意図から直結した提言**のみを書く。意図と関係ない一般論的提言は禁止。`sub_questions` の観点がすべて結論章で回答されているかを執筆者メモで自己点検 |

**⚠️ 問い/答え形式の禁止（再掲）**: サブクエスチョン（SQ1 等）はあくまで執筆者の内部メモであり、レポート本文には通常の宣言調の論述を書く（詳細: `analysis/terminology.md` §5-A-2）

**記述例（タイプ B: 3 社 EV 戦略比較、視座は「各社のポジション差異を明らかにする」）**:

> 本母集団（3 社合計 2,341 件）では、トヨタが 52%、ホンダが 28%、日産が 20% を占める（L1）。特にトヨタは H01M8（燃料電池）にも 12% の出願をしており、電池駆動 EV に集中するホンダ・日産とは異なる広域な電動化戦略を示唆する（L2）。本分析の視座である「3 社のポジション差異」に照らすと、これはトヨタの技術ポートフォリオが最も分散していることを意味し、競合 2 社が集中する H01M2（リチウムイオン電池）領域での競争では相対的に手薄である可能性がある（L3）。従って、3 社比較の観点からは、H01M2 領域を主戦場と見るか広域電動化全体を対象と見るかで、競争構造の見え方が大きく変わる。提言としては、調査対象社が H01M2 領域での優位を確立したい場合、特許出願の集中投下を短期で進める必要がある（L4）。

### 🎯 母集団タイプ（A/A'/B/C/D）別の 4 層モデル運用

Phase A STOP-GATE C で確定した母集団タイプ（`reports/_phase_a_decisions.json` に記録）に応じて、**4 層モデルで使える指標・使える表現が変わる**。詳細は `analysis/population_type_metrics.md` §2 の指標解釈表と §3 のタイプ別禁止表現リストを参照。

特に注意すべきポイント:

| タイプ | Layer 1（事実） | Layer 3（洞察） の制約 |
|---|---|---|
| **A 業界全体 / A' 技術領域** | 業界/技術領域の数値として解釈可 | 業界全体への一般化が可能（Web 裏付けの条件付き） |
| **B 競合限定** | 「**調査対象 N 社内での**」限定必須 | 「市場集中」「業界寡占」等の表現は**禁止**。「対象社間の非対称性」に限定 |
| **C 単一企業** | 「**当社の**」限定必須。出願人 HHI は算出しない | 「市場」「業界」を含む全表現が**禁止**。発明者集中度・IPC 集中度で代替 |
| **D 特定製品・技術テーマ** | 「本テーマでは」限定必須 | 「業界全体では」は禁止。テーマ内の構造として記述 |

**NG 例（タイプ C で市場解釈の誤用）**:

> 本分析では出願人 HHI = 1.0 を示し、市場は極めて寡占的である（L1/L2）。これは業界が単一プレイヤーに支配されていることを示す（L3）。

❌ タイプ C では HHI は常に 1.0 で意味を持たない。市場・業界解釈は母集団に含まれないため禁止。

**OK 例（タイプ C で適切な指標・解釈）**:

> 当社の特許群（N 件）では、発明者集中度 HHI = 0.08 と分散しており、複数のチームが並行して R&D を推進していることが示唆される（L1/L2）。IPC 分布では H01M* が 42% を占め、当社の主力領域であることが確認できる（L2）。外部市場調査によれば#footnote[...]、同領域の市場規模は年 12% 成長しており、当社の注力投資は市場成長と整合的である（L3）。従って、H01M 領域への R&D 投資を次年度も維持・拡大することを推奨する（L4）。

→ 詳細ルール: `analysis/population_type_metrics.md`

**不合格パターン（頻出）:**
- L1の事実列挙だけで1セクションが終わる（クラスタ一覧、出願人ランキング等）
- L2の解釈が1行（「〜を示す」「〜を反映している」で終わる）
- L3が完全に欠落（最も多いパターン）
- L4が汎用的（「今後の動向を注視すべき」「参入を検討すべき」等の抽象表現）

**合格パターン:**
- L1: 「クラスタ0（タイヤ用ゴム、14件）のCAGR +254.5%」
- L2: 「2021年以降の急増はタイヤメーカー（TOYO TIRE）の参入と旭化成のマスターバッチ技術確立が背景」
- L3: 「このCAGR +254.5%はCNF全体の-23.0%と真逆であり、技術の主戦場が基盤材料からゴム応用に構造的に転換していることを意味する。製紙大手はこの転換に乗り遅れている」
- L4: 「製紙大手がこの領域に参入するには、タイヤメーカーとの共同開発契約を2025年中に締結し、CNF×ゴムの配合最適化技術を内製化する必要がある」

## 2. 数値根拠の標準書式

全ての主張には具体的な数値根拠を付すこと。

### 件数・割合
- 「A社は152件（全体の23.4%）」— 件数と割合を必ず併記
- 「上位3社で全体の67.2%を占める」— 集中度を示す

### 成長率
- 「CAGR +15.2%/年（2018→2023年、32件→65件）」— 率・期間・始終値を全て含む
- 成長が見られない場合も明記:「CAGR -2.1%/年（停滞期）」

### 集中度
- 「HHI = 0.32（高集中型: 上位2社寡占）」— 値 + 解釈区分
- 解釈基準: < 0.15 分散型 / 0.15-0.25 中程度 / > 0.25 高集中型

### 多様性指標の解釈基準

| 指標 | 算出方法 | 解釈 |
|------|--------|------|
| HHI | Σ(share²) | < 0.10: 競争的、0.10-0.18: 中程度、> 0.18: 寡占 |
| Entropy | -Σ(p × log₂(p)) | 高い=多様（分散）、低い=集中 |
| Gini | 累積分布の不平等度 | 0=完全平等、1=完全不平等 |

3指標を組み合わせることで、単一指標では捉えられない市場構造の特徴を立体的に把握できる。
例: HHI低（競争的）× Gini高（不平等）= 上位少数が支配しつつも多数の小規模プレイヤーが参入している状態。

### ネットワーク指標
- 「媒介中心性 0.45（ネットワーク内最高値）」— 値 + 相対位置
- 「ネットワーク密度 0.18（中程度: 協業は活発だが閉じたクラスタが存在）」

### 期間
- 「直近3年（2021-2023年）」— 常に具体的な年を併記
- 「分析対象期間: 2015-2023年（9年間、総計2,340件）」

## 3. 段落構成のパターン

1つのEvidence内の各段落は以下の4文構成を基本とする。

```
[主張文] クラスタ3は半導体製造プロセス革新の最前線である。
[数値根拠] 152件（全体の23.4%）を含み、CAGR +15.2%/年で成長中。A社（45件）とB社（32件）が牽引している。
[特許引用] 代表的なA社特許（特開2022-XXXXXX「次世代EUVリソグラフィ装置」）は同クラスタの技術的方向性を端的に示している。
[含意] この集中は、EUVリソグラフィが次世代半導体製造の標準技術として確立されつつあることを意味する。
```

- 段落の長さ: 4-8文（短すぎず長すぎず）
- 1段落1主題: 1つの段落で複数のクラスタや複数のテーマを混在させない
- 接続: 段落間の論理的つながりを接続詞で明示する（「一方」「これに対し」「さらに」「総合すると」）

## 4. レポート品質チェックリスト

レポート完成後（SKILL.md Phase D）、以下を全て確認すること。不合格項目がある場合はレポートを修正してから出力する。

### 必須要件（全10項目合格が必要）
- [ ] **全モジュールdeep_dive**: `reports/` に最低4つの `*_deep_dive.typ` が存在するか（Saturn V, Explorer, MEGA, ATLAS）。各ファイルの最低行数を満たしているか（Saturn V 200行, Explorer 200行, MEGA 120行, ATLAS 120行, CORE 80行）。NEBULAデータが存在する場合は `nebula_deep_dive.typ`（80行以上）も必須。不合格の場合はPhase 5aに戻る
- [ ] **ミクロ分析**: 全deep_diveにミクロ分析A（代表特許引用）とミクロ分析B（企業戦略プロファイル）が含まれているか。代表特許は公開番号・タイトル・出願人の3点を含む形式で引用されているか。各deep_diveのミクロ分析Aで合計15件以上の代表特許が引用されているか
- [ ] **情報ロスチェック**: report.typの各モジュール章の行数が、対応するdeep_dive.typの行数の90%以上であるか（`wc -l` で両ファイルの該当行数を比較）。deep_diveの全セクション（ブリッジ分析、ホワイトスペース分析、バリューチェーン分析、ミクロ分析等）がreport.typに全て含まれているか
- [ ] **数値根拠**: 全ての主張に具体的な数値が含まれているか
- [ ] **代表特許引用**: 各deep_diveのミクロ分析Aで合計15件以上の代表特許が、公開番号・タイトル・出願人の3点セットで引用されているか
- [ ] **クロスモジュール分析**: 最低3つのクロスモジュール分析パターン（cross_module.md参照、P1-P13）が実施されているか
- [ ] **事実と推論の分離**: 4層モデルの各層が明確に区別されているか
- [ ] **データソース明示**: 全evidence-boxに具体的なモジュール名・分析手法名を含むデータソースマーカー（「Saturn V TELESCOPE分析の結果」「Explorerの共起ネットワーク分析によると」「MEGA PULSE分析（CAGR×活動量）の結果」等）が付与されているか。「マクロ統計データによると」等の抽象的表現は不可（→ セクション5参照）
- [ ] **スナップショット画像（全章必須）**: **レポートの全ての章（= 見出し単位）に最低1つの `#snapshot-figure()` が含まれていること**。チャート・マップの視覚的根拠がない章は品質不合格とする。`ls snapshots/` で利用可能な画像を全て確認し、各章に最適な画像を割り当てる。特に技術ランドスケープ、クラスタ動態マップ、出願トレンド、共起ネットワーク、MEGA 4象限、Hype Cycleは必須挿入。同一画像を複数章から異なる分析視点で引用可
- [ ] **AIインサイト反映**: `prompts/` から読み込んだAIインサイトの分析内容（マップの空間的読み取り、クラスタの技術的解釈、プレイヤー動態等）がレポートのEvidence分析に反映されているか。`prompts/` のファイル命名規則: `atlas_trend_insight.md`, `atlas_ranking_insight.md` 等（voyager_schema.md参照）
- [ ] **NEBULA環境コンテキスト**: NEBULAデータが存在する場合、(1) `nebula_deep_dive.typ` がレポート冒頭（エグゼクティブサマリー直後）に配置されているか、(2) 各モジュールのdeep_diveでNEBULA環境分析の主要仮説が参照されているか
- [ ] **ノイズ分析**: Saturn V deep_diveに以下の4セクションが全て含まれているか（`analysis/noise_analysis.md` の5手法フレームワークに基づく）: (1) ノイズ率の戦略的解釈（閾値表: 5%以下=成熟, 5-15%=標準, 15-30%=多様, 30%超=発散）、(2) 空間分布分析（UMAP座標に基づくパターン分類: 周辺散在/集中/孤立）、(3) 出願人分析（ノイズ特許の企業別集中度）、(4) 萌芽領域抽出（ノイズから2つ以上の新興テーマを命名・件数付きで特定）。exemplarでは約50行のノイズ分析セクションを含んでおり、同等の深さを求める
- [ ] **クラスタ動態マップの4象限解釈**: Saturn V/EAGLE/NEBULAのクラスタ動態マップ（累積件数xCAGR）が4象限で解釈されているか
- [ ] **多様性指標（Entropy/Gini）の解釈**: ATLAS deep_diveにHHI・Entropy・Giniの3指標組み合わせ分析が含まれているか（セクション2「多様性指標の解釈基準」参照）
- [ ] **学術-特許クロス分析**（NEBULAデータがある場合）: NEBULA学術クラスタ vs Saturn V特許クラスタのクロス分析（cross_module.md P13参照）が実施されているか

### 段落深度の定量チェック

品質検証時、report.typの各章（= 見出し）について以下を確認する:

- 1行だけの箇条書きが5行以上連続していないか → 不合格（段落に展開すること）
- 「〜を示す。」「〜を反映している。」で終わる1文段落が3つ以上連続していないか → 不合格（L3/L4を追加すること）
- 各出願人の記述が1行以内で終わっていないか → 不合格（5行以上に展開すること）
- クロスモジュール分析の各パターンが5行以内で終わっていないか → 不合格（15-20行に展開すること）

### 推奨要件

## 5. データソース明示ルール（ハルシネーション防止 / 用語統一版）

レポート中のデータ引用には、**具体的なモジュール名・分析手法名**を明示するマーカーを必ず付与する。「マクロ統計データによると」のような抽象的表現は禁止し、読者がデータの出自を正確に追跡できるようにする。

**マーカー中に内部ファイル名（`*.json`）・内部フィールド名（`spatial_context`, `cluster_dynamics`, `quadrant_summary`, `time_series` 等）・内部ガイドファイル名（`*.md`）を書くことを禁止する。** 必ず `analysis/terminology.md` の正式日本語呼称に置き換えること。

### 適用ルール

1. **モジュール名は必須**: 全てのデータ引用には APOLLO のモジュール名（Saturn V / Explorer / MEGA / ATLAS / CREW / EAGLE / NEBULA / CORE）を含めること。「マクロ統計データ」「集計データ」等の抽象表現は使用禁止
2. **分析手法名を併記**: 可能な限り、分析手法（SBERT + UMAP + HDBSCAN、Jaccard 共起、CAGR × 活動量 4 象限等）を併記する
3. **特許データセットを直接参照した場合**: フィルタリング・検索した結果を述べる際には、検索条件（「クラスタ 3 に絞込」「A 社の出願のみ」等）を**自然な日本語で**含めたマーカーを付与する。`cluster==3` のようなコード片を本文に出してはいけない（付録のメソドロジー節でのみ許容）
4. **クロスモジュール照合は必須**: 複数モジュールの結果を照合した箇所は、両方のモジュール名を必ず明示する
5. **マーカーの自然な挿入**: 文頭・文中いずれでも可。文章の自然さを損なわない位置に挿入する
6. **⚠️ 絶対禁止**: 内部ファイル名（`*.json`）・内部フィールド名（`spatial_context`, `cluster_dynamics`, `quadrant_summary`, `time_series`, `hype_cycle_phase`, `noise_analysis`, `macro_events`, `npl_gap_analysis` 等）・内部ガイドファイル名（`*.md`）・**内部プロセス用語**（`deep_dive`, `Phase A/B/C/D` 等）をレポート本文に書くこと。`deep_dive` は「モジュール別分析章」「Saturn V 分析章」等に置換する。詳細は `analysis/terminology.md` を参照

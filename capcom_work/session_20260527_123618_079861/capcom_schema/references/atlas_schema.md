# ATLAS スキーマ

## 対象ファイル
`data/atlas_statistics.json`

## モジュール概要
ATLAS (Advanced Trend & Landscape Analysis Statistics) は、APOLLOプラットフォームの基本統計モジュール。特許データの出願件数時系列推移、出願人ランキング、IPC分類分布、市場集中度(HHI)などの俯瞰的な定量指標を提供する。このJSONは、CAPCOMセッションが有効な場合にフィルタ適用後のデータから自動生成され、`capcom.save_data()` によって `data/` ディレクトリに保存される。

## JSONスキーマ

### トップレベル構造

| フィールド名 | 型 | 説明 |
|---|---|---|
| `metadata` | object | 分析の基本メタ情報 |
| `trend` | object | 年別出願件数の辞書 |
| `cagr` | string | 年平均成長率(CAGR)。パーセント表記の文字列 |
| `trend_direction` | string | トレンドの方向性ラベル |
| `applicant_ranking` | object | 出願人ランキング(上位30件) |
| `ipc_ranking` | object | IPCサブクラスランキング(上位20件) |
| `hhi` | number \| null | ヘルフィンダール・ハーシュマン指数(0-1スケール) |
| `hhi_status` | string \| null | HHIに基づく市場集中度の判定ラベル |
| `status_distribution` | object \| null | ステータス別件数（任意。ステータスカラムがある場合のみ） |
| `status_by_year` | object \| null | 年×ステータスのクロス集計（任意） |

### `metadata` オブジェクト

| フィールド名 | 型 | 説明 | 例 |
|---|---|---|---|
| `module` | string | 固定値 `"ATLAS"` | `"ATLAS"` |
| `period` | string | 集計対象期間。`"開始年-終了年"` 形式 | `"2010-2023"` |
| `total_patents` | integer | フィルタ適用後の総特許件数 | `1523` |
| `unique_applicants` | integer | 出願人ランキングに含まれるユニーク出願人数(上位30件の数) | `30` |

### `trend` オブジェクト

| キー | 値の型 | 説明 | 例 |
|---|---|---|---|
| `"年"` (文字列) | integer | その年の出願件数 | `{"2010": 45, "2011": 52, ...}` |

年は文字列型のキーとして格納される。値は整数。期間内の全年を網羅する(欠損年は含まれない場合がある)。

### `cagr` フィールド

| 値 | 説明 |
|---|---|
| `"12.3%"` | パーセント表記の文字列(小数点1桁)。`calculate_cagr_slope()` で算出 |
| `"N/A"` | 計算不能の場合(データ不足等) |

CAGRは期間の最初の年と最後の年の出願件数から `(end/start)^(1/n_years) - 1` で算出される。

### `trend_direction` フィールド

| 値 | 条件 |
|---|---|
| `"急上昇"` | 線形回帰の傾き > 0.5 |
| `"増加傾向"` | 0 < 傾き <= 0.5 |
| `"減少傾向"` | -0.5 < 傾き <= 0 |
| `"失速"` | 傾き <= -0.5 |
| `"不明"` | 回帰計算失敗時 |
| `"Stable"` | データ点が2未満の場合 |
| `"N/A"` | 値が `None` の場合 |

### `applicant_ranking` オブジェクト

| キー | 値の型 | 説明 | 例 |
|---|---|---|---|
| `"出願人名"` (文字列) | integer | その出願人の出願件数 | `{"トヨタ自動車": 120, "本田技研": 85, ...}` |

上位30件まで格納。降順にソートされている。出願人名は `applicant_main` カラム(リスト型)を展開・集計したもの。

### `ipc_ranking` オブジェクト

| キー | 値の型 | 説明 | 例 |
|---|---|---|---|
| `"IPCサブクラス"` (文字列) | integer | そのIPCサブクラスに属する件数 | `{"H04L": 230, "G06F": 185, ...}` |

上位20件まで格納。IPCコードは先頭4文字(サブクラスレベル)に切り詰められている。区切り文字は `st.session_state['delimiters']['ipc']` で制御される。

### `hhi` フィールド

| 値 | 説明 |
|---|---|
| `0.0523` | HHI値(0-1スケール、小数点4桁に丸め) |
| `null` | 出願人データなし、または計算不能 |

HHI (Herfindahl-Hirschman Index) は `applicant_ranking` の件数分布から算出。各出願人のシェア(件数/合計)の二乗和。

### `hhi_status` フィールド

| 値 | 条件 |
|---|---|
| `"競争的 (分散)"` | HHI < 0.10 |
| `"中程度の集中"` | 0.10 <= HHI < 0.18 |
| `"寡占的 (高集中)"` | HHI >= 0.18 |
| `"データ不足"` | 出願人データなし |
| `null` | `hhi` が `null` の場合 |

公正取引委員会の基準に準拠した判定。

### `status_distribution` オブジェクト（任意）

| キー | 値の型 | 説明 | 例 |
|---|---|---|---|
| `"ステータス名"` (文字列) | integer | そのステータスの件数 | `{"登録": 1450, "公開": 980, ...}` |

ステータスカラム（`col_map['status']`）が存在する場合のみ出力。特許の法的状態（登録、公開、拒絶、取下等）別の件数分布。

### `status_by_year` オブジェクト（任意）

| キー | 値の型 | 説明 | 例 |
|---|---|---|---|
| `"年"` (文字列) | object | 年ごとのステータス別件数 | `{"2015": {"登録": 120, "公開": 60}, ...}` |

年×ステータスのクロス集計。各年のステータス内訳を辞書で格納。

## 解釈ガイドライン

### データの読み方

1. **`trend` の解釈**: 年別出願件数は「技術分野全体の活動量」を示す。急増年は技術ブームや規制変更を示唆し、急減年は技術成熟・代替技術の台頭を意味することが多い。ただし、直近1-2年は特許公開の遅延(出願から公開まで通常18ヶ月)により実態より少なく見える点に注意が必要。

2. **`cagr` と `trend_direction` の関係**: CAGRは期間全体の平均成長率であり、`trend_direction` は線形回帰の傾きに基づく。両者が矛盾する場合(例: CAGRはプラスだが trend_direction は「減少傾向」)、期間の前半に急成長があり後半に減速した可能性がある。

3. **`applicant_ranking` の注意点**: 出願人名は前処理で正規化されているが、同一企業の別名義(子会社、旧社名等)が分離されている場合がある。上位30件に限定されているため、ロングテールの零細出願人は含まれない。

4. **`ipc_ranking` の粒度**: サブクラスレベル(4桁)での集計。1件の特許が複数のIPCを持つ場合は重複カウントされるため、合計は `total_patents` を超えることがある。

5. **`hhi` と `hhi_status` の解釈**: HHIは上位30出願人のみから算出されるため、全出願人を対象とした場合より高めに出る傾向がある。`hhi_status` の判定閾値は公正取引委員会の基準(0-1スケール)に準拠。

6. **`status_distribution` の解釈**: 登録率（登録件数/総件数）は技術分野の成熟度を示す。登録率が高い分野は権利化しやすく、低い分野は審査が厳しいか出願戦略が防衛的であることを示唆する。拒絶率・取下率が高い場合は、出願の質や戦略に問題がある可能性がある。

7. **`status_by_year` の解釈**: 年別のステータス推移は、審査動向の変化や出願戦略の変遷を把握するのに有用。直近年は審査未了（公開状態）の件数が多くなる点に注意。登録率の年次推移が低下している場合、審査基準の厳格化や技術分野の成熟を示唆する。

### null値の取り扱い

- `hhi` / `hhi_status` が `null`: 出願人ランキングが空の場合。データそのものが存在しないことを意味する。
- `cagr` / `trend_direction` が `"N/A"`: 計算に十分なデータがない場合(1年分のデータのみ等)。

## レポートでの典型的な言及パターン

### 市場規模・トレンド概況
- 「対象期間 {period} における総出願件数は {total_patents} 件であり、CAGR {cagr} で {trend_direction} を示している」
- 「{trend の最大年} 年にピーク({件数}件)を迎え、以降は {増減パターン} の傾向にある」

### 競争環境分析
- 「HHI {hhi} ({hhi_status}) であり、市場は {集中度の評価} の状態にある」
- 「トップ出願人 {1位名} は {1位件数} 件({シェア%})を占め、2位の {2位名}({2位件数}件)との間に {倍率} 倍の差がある」

### 技術分野の多様性
- 「IPC分布を見ると、{1位IPC} ({件数}件) が最大で、{2位IPC}、{3位IPC} が続く。上位3分野で全体の約{割合}%を占める」

### クロスリファレンス
- `trend` の変曲点と MEGA モジュールの CAGR 分析を照合することで、特定出願人がトレンドの変動にどの程度寄与しているかを特定可能
- `applicant_ranking` と CREW モジュールの共著ネットワークを組み合わせることで、上位プレイヤー間の連携関係を可視化可能
- `ipc_ranking` と Saturn V のクラスタリング結果を対比することで、IPC分類では見えない技術的類似性を発見可能

## 新規フィールド: 多様性指標

`atlas_statistics.json` に以下のフィールドが追加されている:

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `entropy` | float | Shannon エントロピー（高い＝分散、低い＝集中） |
| `gini` | float | Gini係数（0＝完全平等、1＝完全不平等） |
| `n_entities` | int | 非ゼロエンティティ数 |

### 3指標の統合解釈ガイド

HHI・Entropy・Giniの3指標は単独では一面的。組み合わせることで市場構造の立体的把握が可能:

| パターン | HHI | Entropy | Gini | 解釈 |
|---------|-----|---------|------|------|
| 少数寡占 | 高(>0.18) | 低 | 高(>0.5) | 上位数社が支配。参入障壁が高い |
| 分散競争 | 低(<0.10) | 高 | 低(<0.3) | 多数の小規模プレイヤーが競合。標準化前段階の可能性 |
| 二極化 | 中 | 中 | 高(>0.6) | 上位と下位の格差が大きい。中堅の存在感が薄い |
| 均質分散 | 低 | 高 | 低 | 参入者が多く均等に分布。成熟市場 or 黎明期 |

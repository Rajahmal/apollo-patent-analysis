# Phase B STOP-GATE 2 — Web調査可否確認テンプレ

SKILL.md §Phase B STOP-GATE 2 の `ask_user_question` 呼び出し用雛形。

---

## 前提

Mission Objective と Phase A で把握したデータ特性を元に、**Web調査テーマを3-5件** 事前に導出しておく。例：
- 「主要出願人の直近プレスリリースから新製品ロードマップを確認」
- 「市場規模・CAGR（調査会社レポート）」
- 「政策動向（経産省/NEDO の戦略文書）」

---

## ask_user_question 呼び出しフォーマット

```yaml
question: "以下のテーマで Web 調査を実施しますか？"
header: "Web調査"
multiSelect: false
options:
  - label: "実施する（推奨）"
    description: "以下の3-5テーマで Web 検索を行い、レポートの外部情報として活用する"
  - label: "スキップ（特許データのみで書く）"
    description: "Web 調査を省略。レポートは特許データ内の数値根拠のみで構成する"
  - label: "テーマを修正したい"
    description: "提示したテーマから追加/削除したい。修正指示を Other で入力"
```

## 提示するWeb調査テーマ（ask_user_questionの前にテキストで提示）

`ask_user_question` 呼び出しの **直前** に、以下のようにユーザーへテーマ一覧を提示する：

```
Phase B Web 調査テーマ候補:
1. <出願人名> の2024-2025年プレスリリース（新製品発表・研究開発動向）
2. <技術分野> 市場規模予測（調査会社レポート: 富士経済 / 矢野経済 等）
3. <政策名> 政策動向（経産省「次世代電池戦略」等、国の補助金・規制）
4. 主要学会の最新発表（<学会名>）
5. 競合国外企業の参入動向
```

その直後に `ask_user_question` を呼び出して実施可否を取得する。

---

## ゲート通過後の処理

| ユーザー回答 | 次のアクション |
|---|---|
| 「実施する」 | Phase B 本体作業の中で Web 検索を実行。出所（URL・サイト名・取得日）を必ず記録 |
| 「スキップ」 | Web 調査は省略。レポートには特許データ内の数値のみを根拠として使う |
| 「テーマを修正したい」 | 修正内容を別の `ask_user_question` で確認してから再度可否を聞く |

---

## 禁止事項

- **「不要と判断しました」等の AI 自己判断でスキップするのは禁止**（SKILL.md L138）
- 必ずユーザーの明示的な応答を取得する
- 「実施する」と回答された場合、後続レポートで Web 情報を使う際は必ず出所を明記する（`phase_d_gate.sh` の Check 6 で検証される）

---

## 出所表記の例（Web情報をレポートに引用する場合）

Typst:
```typst
#footnote[出所: 富士経済「2024年版 次世代電池市場の全貌」(取得日: 2026-04-15)]
```

Markdown:
```
主要市場調査によれば CAGR は +8.3%/年 である[^1]。

[^1]: 出所: 富士経済「2024年版 次世代電池市場の全貌」(取得日: 2026-04-15)
```

`phase_d_gate.sh` の Check 6 は「Web情報使用件数 > 出所記載件数」の場合に FAIL する。必ず1対1対応で出所を付けること。

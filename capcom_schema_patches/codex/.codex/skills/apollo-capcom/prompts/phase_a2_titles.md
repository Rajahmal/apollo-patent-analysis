# Phase A-2 — タイトル3案選択用 ask_user_question テンプレート

SKILL.md §Phase A-2 の STOP-GATE で使用。`ask_user_question` ツールに渡す質問項目の雛形。

---

## 使い方（実行手順）

1. Phase A で把握した **Mission Objective** と **データ特性**（出願人上位、クラスタ数、期間等）を踏まえて、以下のフォーマットで **3案** を生成する
2. 生成した3案を `ask_user_question` に渡してユーザーに選択させる
3. ユーザーが選択した案（または「Other」で指定された案）を採用し、`voyager/mission.json` に `confirmed_title` として記録

---

## 生成ルール

### タイトル
- **オーソドックス**（標準的・保守的）な体言止め。**20 文字以内**の目安
- **問いかけ型は禁止**（例: ×「どこへ向かうのか？」）
- **扇情的・文学的タイトルは禁止**（例: ×「独断 — 電池の未来」「電池、敗北の構造」）
- 指針: 「{技術分野 / 対象企業} の {分析種別}」の単純な組み合わせが基本。クリエイティブなコピーは不要
  - OK 例: 「CNF 特許動向分析 2026」「全固体電池の競合ポジション分析」「次世代半導体製造技術ランドスケープ」

### サブタイトル
- **30文字以内**
- **具体的な件数・期間・分析軸**を含める
- 例: 「N件の特許から見る2015-2024の技術動向」

---

## ask_user_question 呼び出しフォーマット

```yaml
question: "採用するレポートタイトルを選択してください。各案はタイトル+サブタイトル構成です。"
header: "タイトル選択"
options:
  - label: "<15文字以内のタイトル案1>"
    description: "<30文字以内のサブタイトル案1、根拠メモ付き>"
  - label: "<15文字以内のタイトル案2>"
    description: "<30文字以内のサブタイトル案2、根拠メモ付き>"
  - label: "<15文字以内のタイトル案3>"
    description: "<30文字以内のサブタイトル案3、根拠メモ付き>"
```

（Codex の `ask_user_question` は「Other」自動追加のため、自前で Other を含めない）

---

## 具体例（参考）

**Mission Objective**: 「全固体電池の競争環境と技術トレンドを可視化」
**データ**: 特許 3,214件、2015-2024、Saturn V クラスタ 12個

```yaml
question: "採用するレポートタイトルを選択してください。各案はタイトル+サブタイトル構成です。"
header: "タイトル選択"
options:
  - label: "全固体電池 特許動向分析 2024"
    description: "3,214件の特許から読み解く2015-2024年の技術動向"
  - label: "全固体電池技術の競合ポジション分析"
    description: "12 クラスタと主要出願人の競争構造を俯瞰"
  - label: "全固体電池技術ランドスケープ"
    description: "2015-2024 年・3,214 件を対象とした特許・競合分析"
```

---

## 禁止事項

- **3案生成をスキップして「1案だけ提示してよいか」を聞くのは禁止** (SKILL.md §0 第5項)
- **AI側で勝手に1つに絞るのは禁止** (SKILL.md L108-112 参照)
- 必ず `ask_user_question` を呼び出して応答を待つ。テキスト出力だけで進めてはならない

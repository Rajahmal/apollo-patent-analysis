# CREW スキーマ

## 対象ファイル
`data/crew_network.json`（要約版）

## モジュール概要
CREW は発明者/出願人の共起ネットワーク分析モジュール。特許の共同発明者・共同出願人の関係をグラフ構造として抽出し、媒介中心性（betweenness centrality）によるキーパーソン特定、Louvain法によるコミュニティ検出、技術ブローカー分析を行う。

## JSONスキーマ

### metadata

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `module` | string | 固定値 `"CREW"` |
| `mode` | string | `"発明者ネットワーク"` または `"出願人ネットワーク"` |
| `n_nodes_total` | int | 元データの全ノード数 |
| `n_nodes_exported` | int | エクスポートされたノード数（上限50） |
| `n_edges_total` | int | 元データの全エッジ数 |
| `n_edges_exported` | int | エクスポートされたエッジ数（上限200） |
| `density` | float | ネットワーク密度（0.0〜1.0）。全ノード間の可能なエッジ数に対する実際のエッジ数の比率 |
| `n_communities` | int | 検出されたコミュニティ数 |

### nodes（配列）— betweenness降順top50

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | string | ノード名（発明者名または出願人名） |
| `patent_count` | int | 当該ノードが関与する特許数 |
| `degree` | int | 接続エッジ数（直接の共起相手の数） |
| `betweenness` | float | 媒介中心性（0.0〜1.0）。ネットワーク内の最短経路上に位置する頻度 |
| `community` | int | 所属コミュニティID |
| `topics` | string | 当該ノードの関与特許から抽出された主要技術トピック |
| `rising_score` | float | 急上昇スコア。近年の活動量増加率を示す指標 |
| `brokerage` | float | ブローカースコア。異なるコミュニティ間を橋渡しする度合い |

### edges（配列）— weight降順top200

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `source` | string | エッジの始点ノード名 |
| `target` | string | エッジの終点ノード名 |
| `weight` | int | 共起回数（共同で関与した特許の数） |

### communities（配列）— 要約版

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | int | コミュニティID |
| `size` | int | コミュニティ内のノード数 |
| `top_members` | [string] | betweenness上位5名のメンバー名リスト |
| `total_patents` | int | コミュニティ全体の関与特許数（重複含む） |

## データ要約について
- **ノード**: betweenness降順top50に制限。`metadata.n_nodes_total` で元データの全体規模を確認可能。上限を超えるノードは切り捨てられるため、末端の低中心性ノードはエクスポートに含まれない。
- **エッジ**: weight降順top200に制限。`metadata.n_edges_total` で元データの全体規模を確認可能。低頻度の共起関係は切り捨てられる。
- **コミュニティ**: 各コミュニティのbetweenness上位5名 + サイズの要約版。全メンバーリストは含まない。コミュニティの全容把握には元データの参照が必要。

## 解釈ガイドライン

### 媒介中心性（betweenness）
- 0.0〜1.0の範囲。値が高いほどネットワーク内で情報・技術の橋渡し役として重要な位置にいることを示す。
- betweennessが高いノードは、異なる技術グループや組織間の接点となるキーパーソン/キー組織である可能性が高い。
- 特許数（patent_count）が少なくてもbetweennessが高い場合、少数精鋭で戦略的な位置にいることを意味する。

### コミュニティ
- Louvain法により検出された密結合グループ。同一コミュニティ内のノードは互いに強く結びついている。
- コミュニティは技術テーマ、組織系列、研究グループなどに対応することが多い。
- `communities.top_members` は各コミュニティの中核人物/組織を示す。

### 技術ブローカー（brokerage）
- 異なるコミュニティ間を橋渡しするノードの度合いを定量化したスコア。
- ブローカースコアが高いノードは、複数の技術領域やコミュニティを横断して活動している。
- betweennessとbrokerageの両方が高いノードは、ネットワーク全体のハブとして特に注目すべき存在。

### 急上昇スコア（rising_score）
- 近年の活動量増加率を示す指標。正の値が大きいほど、最近急速に活動を拡大していることを意味する。
- 新興プレイヤーの発見に有用。patent_countは少なくてもrising_scoreが高い場合、今後の注目対象となりうる。

### ネットワーク密度（density）
- 0.0に近い場合: 疎なネットワーク。特定のグループ間でのみ共起が発生しており、全体的な協業は少ない。
- 1.0に近い場合: 密なネットワーク。多くのノードが互いに共起しており、広範な協業関係がある。
- 一般的に特許ネットワークは疎（density < 0.1）になりやすい。

## レポートでの典型的な言及パターン

### ネットワーク構造の概観
- 「共起ネットワークは**N名**のノードと**M本**のエッジで構成され、密度は**D**である」
- 「Louvain法により**K個**のコミュニティが検出された。最大コミュニティは**X名**で構成される」
- 「ネットワーク密度が**D**と低いことから、特定のグループ内での共同研究が中心であり、グループ横断的な協業は限定的である」

### キーパーソン/キー組織の特定
- 「媒介中心性が最も高いのは{id}（betweenness: **B**, patent_count: **P**件）であり、ネットワーク全体の情報ハブとして機能している」
- 「{id}はbetweennessとbrokerageの両方が高く、コミュニティ{A}と{B}を橋渡しする技術ブローカーとしての役割を果たしている」
- 「{id}はpatent_countが**P**件と少ないものの、rising_scoreが**R**と高く、近年急速に活動を拡大している新興プレイヤーである」

### コミュニティ間の関係
- 「コミュニティ{A}（{top_members}を中心とする**S名**のグループ）とコミュニティ{B}は、{broker_id}を介して接続されている」
- 「コミュニティ{A}は{topics}に関する技術テーマを共有するグループであり、組織的にも{共通点}が見られる」
- 「孤立したコミュニティ{C}は独自の技術領域で活動しており、他グループとの協業拡大の余地がある」

### 時系列的な変化の示唆
- 「rising_score上位のノードは{id_1}, {id_2}, {id_3}であり、いずれもコミュニティ{X}に属している。この領域での活動活発化が示唆される」
- 「従来の中心的プレイヤーである{id}のrising_scoreは低く、近年の活動は停滞傾向にある」

# EAGLE スキーマ

## 対象ファイル
`data/eagle_clusters.json`

## モジュール概要
EAGLE は探索的ランドスケープモジュール。投げ縄(Lasso)選択による手動クラスタリングを行い、ユーザーの知見を反映したクラスタを作成する。Saturn V の自動クラスタリング(HDBSCAN)とは異なり、ユーザーがUMAP 2D空間上で視覚的にグループを定義する。分析者の戦略的観点・ドメイン知識が直接反映されるため、ビジネス文脈に即した分類となる。

## JSONスキーマ

### metadata

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `module` | string | 固定値 `"EAGLE"` |
| `mode` | string | 固定値 `"manual_lasso"` — 投げ縄による手動選択モードを示す |
| `n_clusters` | int | ユーザーが定義したクラスタ数（未分類を除く） |
| `noise_count` | int | 未分類の特許数（`eagle_cluster == -1`、ユーザーが選択しなかった特許） |
| `total_patents` | int | 分析対象の全特許数 |

### clusters（配列）

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `cluster_id` | int | クラスタID（0始まりの連番。-1は含まれない） |
| `label` | string | ユーザーが付与したクラスタラベル。ラベル編集機能で自由に命名可能。未編集時は `"Cluster {id}"` |
| `count` | int | クラスタ内の特許数 |
| `centroid` | [float, float] | UMAP 2D空間の重心座標 `[umap_x, umap_y]`（小数点以下4桁） |
| `representative_patents` | [string] | 代表特許の要約テキストのリスト。各要素は `"  * [{pub_number}]【{title}】({applicant}, {year}, IPC:{ipc}): {abstract_200chars}..."` 形式。SBERT埋め込みのクラスタ重心に最も近い最大3件を選定 |

### トップレベル

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `spatial_context` | string | UMAP空間におけるクラスタ配置の自然言語記述。`utils_spatial.py` により生成される。各クラスタの近傍クラスタ（上位2件）の関係を含む。空間データが存在しない場合は空文字列 |

## 解釈ガイドライン

- **eagle_cluster = -1 は未分類**: ユーザーが投げ縄で選択しなかった特許群。Saturn V のノイズ（HDBSCAN密度閾値以下）とは異なり、「分析者が意図的に分類対象としなかった」ことを意味する。未分類比率が高い場合、分析者が特定の技術領域に焦点を絞った可能性がある。
- **Saturn V のクラスタとは独立した分類体系**: EAGLE のクラスタIDは Saturn V のクラスタIDとは無関係。同じ特許が両方で異なるクラスタに属することがあり得る。Saturn V が統計的な密度ベースの分類であるのに対し、EAGLE はユーザーの主観的判断に基づく分類である。
- **ユーザーの意図（戦略的観点）が反映されている**: ラベル名や分類基準は分析者のドメイン知識に依存する。自動クラスタリングでは捉えにくいビジネス上の区分（例: 自社技術領域 vs 競合技術領域、攻撃対象 vs 防衛対象）が含まれ得る。
- **centroid はUMAP 2D空間の重心座標**: 意味的に近いクラスタは座標上でも近接する。ただしUMAPの非線形変換により、距離の絶対値に厳密な意味はない。相対的な近接関係に着目すること。
- **representative_patents はSBERT重心近傍から最大3件選定**: 各代表特許の要約テキストは200文字で截断されている。公開番号(`pub_number`)が含まれるため、詳細情報が必要な場合は `patents.csv` で検索可能。
- **spatial_context はクラスタ間の空間配置を自然言語で記述**: 各クラスタの最近傍2クラスタの名前が列挙される。技術的関連性の仮説生成に利用できる。

## レポートでの典型的な言及パターン

### ユーザー定義クラスタの概要
- 「分析者がUMAP空間上で**N個**の技術クラスタを手動定義した」
- 「最大クラスタは『{label}』（**X件**, 全体の**Y%**）であり、分析者が最も注目する技術領域である」
- 「未分類特許は**N件**（全体の**X%**）であり、分析者が今回の焦点から除外した技術群を含む」

### クラスタの戦略的解釈
- 「分析者は『{label_A}』と『{label_B}』を区別しているが、UMAP空間上では近接しており、技術的には連続スペクトラムを形成している可能性がある」
- 「『{label}』は他クラスタから空間的に孤立しており、分析者が独立した技術領域として認識していることと整合する」
- 「分析者のラベル命名から、{ビジネス的観点}での分類意図が読み取れる」

### Saturn V との比較
- 「Saturn V の自動クラスタリングでは**M個**のクラスタが検出されたのに対し、EAGLEでは分析者が**N個**に統合・分割した」
- 「Saturn V のクラスタ『{auto_label}』に相当する領域が、EAGLEでは『{eagle_label_A}』と『{eagle_label_B}』に分割されている」
- 「自動分類と手動分類の差異から、分析者が重視する技術境界が明らかになる」

## 新規フィールド: クラスタ動態マップ

`data/eagle_cluster_dynamics.json` が新規出力される:

```json
{
  "cluster_dynamics": {
    "x_axis": "cumulative_count",
    "y_axis": "cagr",
    "cagr_window_years": 5,
    "clusters": [...],
    "quadrant_summary": {...}
  }
}
```

構造は `saturnv_schema.md` のクラスタ動態マップと同一。
EAGLEは手動クラスタリングのため、分析者の仮説に基づくクラスタの成長・衰退を定量的に評価できる。
Saturn Vの自動クラスタ動態と比較することで、AI vs 人間の視点の違いを検証可能。

# 特許引用ガイド

レポート中の主張を具体的な特許で裏付けるための手順・書式をまとめる。

---

## 1. 引用の目的

- **主張の具体化**: 「半導体製造技術が成長中」→ 具体的な特許番号・タイトルで実証
- **読者の追跡可能性**: レポート読者が元データにアクセスして検証できるようにする
- **分析の信頼性向上**: 数値だけでなく具体例があることで説得力が増す

---

## 2. 代表特許の検索手順

### 方法A: JSONのrepresentative_patentsから引用（推奨・最速）

各モジュールのJSONに含まれる `representative_patents` を使う。

```
# saturnv_clusters.json の例
clusters[i].representative_patents = [
  {"title": "...", "applicant": "...", "pub_number": "...", "ipc": "..."},
  ...
]
```

- **利点**: 既にTF-IDF等で選定済み。追加の検索不要
- **注意**: pub_numberが空の場合がある（col_mapにpub_number未設定時）

### 方法B: patents.csvから直接検索（詳細分析時）

pandasで `data/patents.csv` を読み込み、条件検索する。

⚠️ **メモリ節約注意**: `df` 全体を `print()` や表示しないこと。必ず条件フィルタ + 必要カラム選択 + `.head()` で制限して出力する。

```python
import pandas as pd
df = pd.read_csv('data/patents.csv')
# 注意: print(df) や df.to_string() は禁止（コンテキスト溢れの原因）

# クラスタ内の特定出願人の特許（必要カラムのみ + head制限）
target = df[(df['cluster'] == 3) & (df['applicant_main'] == 'A社')][
    ['title', 'pub_number', 'applicant_main', 'year', 'ipc_main_group']
].head(10)

# 直近3年の新規出願（必要カラムのみ + head制限）
recent = df[(df['year'] >= 2021) & (df['cluster'] == 5)][
    ['title', 'pub_number', 'applicant_main', 'year']
].head(15)

# キーワード検索（タイトル・要約）
kw_match = df[df['title'].str.contains('量子ドット', na=False)][
    ['title', 'pub_number', 'applicant_main', 'year']
].head(10)
```

- **利点**: 柔軟な条件検索が可能。特定の出願人・期間・IPC等で絞り込める
- **注意**: この方法で得た結果を記述する際は「**ミクロ分析によると**」マーカーを必ず付与する（→ common_framework.md セクション5参照）

### 方法C: キーワードベースの特許検索

特定の技術用語でタイトル・要約を全文検索する。

```python
# 複数キーワードのAND検索
mask = df['title'].str.contains('リソグラフィ', na=False) & \
       df['abstract'].str.contains('EUV', na=False)
results = df[mask]
```

- **利点**: クラスタ横断で特定技術の特許を発見できる
- **注意**: 検索結果が多い場合は年・出願人で追加絞り込みを行う

## 3. 引用の標準書式

### インライン引用（短い言及）

文中で1件の特許に簡潔に言及する場合:

```
A社の代表特許（特開2022-123456「次世代EUVリソグラフィ装置」）が示すように、...
```

形式: `（公開番号「タイトル」）` — 出願人は文脈で既出の場合は省略可

### ブロック引用（詳細な根拠）

特許の内容を詳しく根拠として使う場合:

```
代表特許の分析:
- **特開2022-123456**「次世代EUVリソグラフィ装置」（A社, IPC: H01L21/027）
  - 要約: XXX技術を用いた高精度パターニング手法を開示
  - 位置づけ: クラスタ3の中核技術であり、同クラスタの技術的方向性を端的に示す
```

### 表形式の引用（複数特許一覧）

クラスタや出願人の代表特許を一覧表示する場合:

```
| No. | 公開番号 | タイトル | 出願人 | IPC | 年 |
|-----|---------|---------|--------|-----|-----|
| 1 | 特開2022-123456 | 次世代EUVリソグラフィ装置 | A社 | H01L21 | 2022 |
| 2 | 特開2021-789012 | 半導体エッチング方法 | B社 | H01L21 | 2021 |
```

## 4. 引用すべき場面

### 必須引用（必ず1件以上の代表特許を含める）

| 場面 | 理由 |
|------|------|
| クラスタの技術内容を説明する時 | 抽象的な説明だけでは読者が技術を具体的にイメージできない |
| 出願人の戦略を論じる時 | 「A社はXXに注力」→ 具体的特許が無いと根拠不明 |
| 新興技術・急成長領域に言及する時 | 「新しい技術が出現」→ どんな特許か示す必要がある |

### 推奨引用（可能な限り含める）

| 場面 | 理由 |
|------|------|
| Explorer急上昇キーワードの説明時 | キーワードの実態を特許で具体化 |
| MEGA 4象限プレイヤー分析時 | 各象限の代表的な技術を示す |
| CREWブローカー分析時 | ブローカーが関わる具体的な技術を示す |
| クロスモジュール分析の結論時 | 複数モジュールの結論を特許で裏付ける |

## 5. pub_numberが無い場合の対処

col_map に pub_number が未設定の場合、公開番号カラムが存在しない。

### 対処方法

1. **app_num（出願番号）で代替**: 「出願番号 2022-XXXXXX」の形式で引用
2. **タイトル + 出願人 + 年で特定**: 番号なしでも最低限「タイトル」「出願人」「出願年」の3点を含める
3. **表記例**: `A社の特許「次世代EUVリソグラフィ装置」（2022年出願）`

番号がない場合でも、特許引用を省略してはならない。タイトル・出願人・年の3点セットは必ず含めること。

## 6. データソース明示ルール（ハルシネーション防止）

特許引用の際、データの取得元を明示すること。

```

```

→ JSONの `representative_patents` から引用したことが文脈で明白なためマーカー不要

→ 「**ミクロ分析によると**」マーカーを冒頭に付与

```
新規12件）と一致し、B社の特開2023-444444「ペロブスカイト太陽電池製造方法」が
この動向を代表している。
```

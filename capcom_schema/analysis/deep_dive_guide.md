## 記述品質の絶対基準（最重要）

**このセクションはdeep_dive執筆の最優先ルールである。行数・セクション数を満たしていても、以下の品質基準を満たさないdeep_diveは不合格。**

### 段落の深度ルール

各トピック（時期区分、クラスタ解説、出願人分析等）について、**最低3段落**で論じること。1行の箇条書きで済ませることは禁止。

**品質不合格の例（絶対に避けること）:**
```
第1期 急成長期（2015-2017年）: 95件→155件。CAGR +27.7%。政策駆動で産学官が一斉に出願を開始。
```

↑ 1行。事実(L1)のみ。なぜ成長したかの因果がない。

**合格品質の例（このレベルを全トピックで維持すること）:**
```
=== 第1期: 成長加速期（2015-2017年、95→155件）

2015年の95件から2017年の155件へ、年平均+27.7%で急成長した。この急成長の
背景には、2014年のナノセルロースフォーラム設立と、同年の「日本再興戦略」
でのCNF技術の明記がある。政策主導の研究開発ブームが産学官を巻き込む形で
出願を加速させた。

この時期の出願人は364社と極めて多様であり、HHI 0.0318（競争的・分散）が
示す通り、探索的な段階にある。製紙大手（日本製紙、大王製紙）が基盤技術
を固める一方、大学（東京大学、京都大学）からの基礎研究出願も目立ち、
技術の成熟度が低い「探索フェーズ」の典型的なパターンを示す。

注目すべきは、この時期に出願された基盤特許の権利継続率が70%を超えている
点である。出願の「量」だけでなく「質」も高かったことを意味し、この時期
の特許群がCNF技術の知財基盤を形成していると評価できる。
```

↑ 3段落。L1(数値)→L2(政策背景の因果)→L2(競争構造の解釈)→L3(権利継続率からの洞察)。

### 4層モデルの実践ルール

各セクションで4層全てを含めること。特にLayer 3（洞察）とLayer 4（提言）が欠落しやすい。

**Layer 3（洞察）の書き方 — 逆説・対比・統合**:
- 「〜にもかかわらず」: 予想と反する事実の組み合わせ
- 「〜と合わせて考えると」: 異なるモジュールの事実を統合
- 「一見〜だが、実際は〜」: 表面と本質の乖離
- 「〜という逆説的構造」: 矛盾する事実から導かれる深い理解

例: 「参入は容易だが事業化は困難」「出願減少は技術の衰退ではなく選択と集中への移行」「HHI低×Giniの高＝分散的だが不平等」

**Layer 4（提言）の書き方 — 具体的アクション**:
- 「〜を検討すべきである」ではなく、「A社は超領域🅰のC0/C5クラスタに注力し、タイヤメーカーとの共同出願を年5件以上に拡大すべきである」
- 数値目標・時間軸・対象企業名を含める

### 「So What?」テスト

全セクションの末尾で自問すること: **「だから何なのか？読者はこの情報でどんな意思決定ができるか？」**

So What? がない例: 「クラスタ0はタイヤ用ゴムで14件、CAGR +254.5%である。」
So What? がある例: 「クラスタ0のCAGR +254.5%はCNF全体のCAGR -23.0%を10倍以上上回り、CNF技術の次の主戦場がゴム複合材料に移行していることを示す。タイヤメーカー（TOYO TIRE等）の参入は、素材メーカーとの共同開発が本格化した証左であり、自社がこの領域に参入するなら2025年が最後の機会となる可能性がある。」

### ミクロ分析の深度ルール

代表特許の引用は「番号＋タイトル＋出願人」の列挙ではなく、**技術的意義と戦略的文脈を1-2文添える**こと。

不合格: `特開2024-161154「ゴム改質用マスターバッチ」（旭化成, 2023年）`

合格: `特開2024-161154「ゴム改質用マスターバッチ」（旭化成, 2023年出願）: CNFをゴム中に高充填分散させる独自技術。タイヤの転がり抵抗低減と耐摩耗性の両立を実現し、CNF×ゴム複合材料分野における旭化成のリーダーシップを象徴する中核特許。`

### 出願人プロファイルの深度ルール

テーブル1行で済ませず、**各社5行以上の戦略分析段落**を書くこと。

不合格:
```
| 旭化成 | 77件 | 6.5% | リーダー | デュアルドメイン（ゴム×樹脂）|
```

合格:
```
**旭化成**（77件、全体シェア6.5%、CAGR +28.5%）

MEGA PULSEでリーダー象限に位置する唯一の化学メーカーであり、全出願人中
最高の成長率を記録している。出願のピークは2022年（24件）で、2021年5件
からのV字回復パターンが特徴的。主力は超領域🅰ゴム・エラストマー複合
（C0タイヤ用ゴム14件、C5マスターバッチ16件）と超領域🅳熱可塑性樹脂複合
（C13ポリアミド50件）のデュアルドメイン戦略。特にゴムマスターバッチ技術
（特開2024-161154）は業界初のCNF高充填配合を実現し、タイヤメーカーとの
共同出願（TOYO TIRE: 特開2025-065789）も増加中。製紙大手が基盤技術で
停滞する中、CNFの応用展開で独自のポジションを確立しつつある。
```

## 共通ルール

### 出力形式
- `reports/` に `*_deep_dive.typ`（Typstフォーマット）として出力する
- `report_style.typ`の関数（`evidence-box`, `styled-table`, `snapshot-figure`, `insight-box`）を使用すること
- 各ファイルの冒頭に `#import "report_style.typ": *` を記述する

### 新規分析要件
- **ノイズ分析**: Saturn V deep_diveにノイズ特許の萌芽テーマ考察を含める
- **クラスタ動態マップ**: Saturn V/EAGLE/NEBULAのクラスタ動態マップ（累積件数xCAGR）の4象限解釈を含める
- **多様性指標**: ATLAS deep_diveにEntropy/Gini指標の解釈を含める（HHIとの3指標組み合わせ分析）
- **学術-特許クロス分析**: NEBULAデータがある場合、NEBULA学術クラスタ vs Saturn V特許クラスタのクロス分析を含める

### 設計意図への明示的参照（v8 全 deep_dive 必須）

**全ての `*_deep_dive.typ` の冒頭段落（最初の evidence-box の前、または導入文として）で、設計意図への参照を 1 文以上含める**。

**目的**: Phase A STOP-GATE で確定した「本分析の視座」および `sub_questions`（作業メモ）を、各 deep_dive で忘れず意識するため。

**実施手順**:

1. **Phase A 決定を読む**: 執筆前に `reports/_phase_a_decisions.json` の `query_intent_summary` と `sub_questions` を確認
2. **この章が扱う SQ を特定**: 各 deep_dive は `sub_questions` のうち特定の 1-3 個の観点を主に扱う。執筆者メモで対応付けを整理（例: Saturn V deep_dive → SQ1, SQ3 を主に扱う）
3. **冒頭段落で意図参照を 1 文**: 本章の分析が本分析の視座にどう貢献するかを宣言調で 1 文示す

**⚠️ 絶対制約**:
- サブクエスチョンは執筆者の内部メモ。**レポート本文に `SQ1` 等の記号、「問い 1」等の表現は書かない**
- 本文は通常の宣言調の論述で書く（詳細: `analysis/terminology.md` §5-A-2）

**記載例**:

❌ **NG（問い/答え形式）**:
> 本章では Q1「最大成長領域はどこか」に答える。A: タイヤ用ゴムが最大成長である。

❌ **NG（作業メモを本文化）**:
> 本章は SQ1 と SQ3 の観点から分析する。

✅ **OK（宣言調の冒頭段落）**:
> 本章は、本分析の視座である「自社の注力領域選定」に即して、Saturn V TELESCOPE 分析の結果から技術領域の成長構造と各社の注力ポジションを読み解く。特に、成長性が顕著な領域と自社未進出の空白領域の識別を主眼とする。

✅ **OK（タイプ B の競合分析 deep_dive 冒頭）**:
> 本章は、本分析の視座である「3 社の EV 戦略比較」に即して、MEGA PULSE 分析の 4 象限サマリから各社のポジションと注力領域の差異を読み解く。3 社の出願動態を並行して確認することで、共通の主戦場と社別の独自路線を識別する。

## ミクロ分析（全deep_dive共通必須セクション）

**なぜミクロ分析が必須なのか**: ミクロ分析がないレポートはマクロ統計の羅列に過ぎず、読者に「具体的にどの特許が重要か」「どの企業が何をしているか」を伝えられない。ミクロ分析こそがレポートの信頼性と実用性を決定する。

全てのdeep_diveファイルに以下のミクロ分析を**必ず**含めること:

### ミクロ分析A（代表特許の詳細引用）

主要セグメント（クラスタ/象限/コミュニティ等）から各3-5件の代表特許を引用する。

**手順**: (1) Evidence JSONの`representative_patents`から候補を取得 → (2) `data/patents.csv`をpandasで条件検索（`df[df['cluster']==N]`等）して詳細を補完 → (3) 以下の形式で引用:

```
- 特開2022-123456「発明タイトル」（出願人名）: この特許は○○技術の△△を実現しており、クラスタの中核をなす。
```

**引用なしの抽象記述（「代表的な特許がある」「重要な技術が含まれる」等）は品質不合格**

### ミクロ分析B（企業別戦略プロファイル）

そのモジュールに関連する**上位5社以上**について分析する。各社について**最低5行**の記述が必要。

**手順**: (1) `patents.csv`の`applicant_main`でフィルタリング → (2) その出願人のクラスタ分布・IPC分布・年別推移を算出 → (3) 以下の項目を全て記述:
- 総出願件数と全体シェア
- CAGR（成長率）とトレンド（増加/減少/V字回復等）
- 主力クラスタ・超領域（どのクラスタに何件出願しているか）
- 競争ポジション（MEGA PULSEの象限位置）
- 技術戦略の特徴と今後の展望

**品質不合格の例（絶対に避けること）**:
```
--- NG ---
旭化成（77件, リーダー, CAGR +28.5%）：ゴム×樹脂のデュアルドメイン戦略。
```
1行では品質不合格。以下のように5行以上で記述すること:
```
--- OK ---
**旭化成**（77件、全体シェア6.5%、CAGR +28.5%）
MEGA PULSEでリーダー象限に位置する唯一の化学メーカー。出願のピークは2022年（24件）で、
V字回復パターンを示す。主力は超領域ゴム・エラストマー複合（C0タイヤ用ゴム14件、C5マスター
バッチ16件）と超領域熱可塑性樹脂複合（C13ポリアミド50件）。特にゴムマスターバッチ技術
（特開2024-161154）は業界初のCNF高充填配合を実現し、タイヤメーカーとの共同出願も増加中。
製紙大手が基盤技術で停滞する中、CNFの応用展開で独自のポジションを確立しつつある。
```

## 環境コンテキスト参照ルール（NEBULA deep_diveが存在する場合）

`reports/nebula_deep_dive.typ` が存在する場合、Step 1-6の各deep_diveで以下を実施すること:
- 冒頭または適切な箇所で、NEBULA環境分析で導出された「外部環境からの主要仮説」を参照し、そのモジュールの分析結果が仮説を支持/棄却するかを論じる
- 例: 「NEBULA環境分析で特定された{年}年の{政策/市場変化}は、本モジュールの分析結果において{クラスタ構造の変化/出願人動態の変化/キーワード出現パターン}として観測された」
- 環境→特許の因果関係を明示的に記述することで、分析の説得力と文脈的な深みを付与する

## Step 0: NEBULA deep_dive（nebula_strategy に応じて分岐）

**⚠️ 最初に `reports/_phase_a_decisions.json` の `nebula_strategy.selected_mode` を確認する**:

| モード | 処理 |
|---|---|
| `execute` | 以下の通常手順で `nebula_deep_dive.typ` を生成 |
| `web_compensation` | `nebula_deep_dive.typ` の代わりに **`external_env_deep_dive.typ`** を生成（Web 調査結果を 4 カテゴリで整理、`#footnote` 引用付き、詳細は後述） |
| `omit` | **Step 0 自体をスキップ**。以降の Step 1-6 で「環境分析結果を参照」する記述も省略 |

### モード `execute` の場合（NEBULA 実行済み）

NEBULAデータ（`nebula_hype_cycle.json` / `nebula_macro_events.json`）が存在する場合、**他の全モジュールに先行して**環境分析を実施する。NEBULA deep_diveは後続のStep 1-6における分析の「外部環境コンテキスト」として機能する。

**手順**:
1. `capcom_schema/exemplars/nebula_exemplar.typ` を読む — **exemplarを読まずに書き始めてはならない**。コメント内の `required_sections` を確認する
2. `references/nebula_schema.md` を読む
3. `data/nebula_hype_cycle.json` と `data/nebula_macro_events.json` を読む
4. **`data/nebula_academic_clusters.json` があれば必ず読む**（学術ランドスケープ）— クラスタ構造、空間配置（spatial_context）、クラスタ動態（cluster_dynamics）、代表論文（representative_papers）を確認する
5. `prompts/nebula_insight_*.md` があれば読む
6. `reports/nebula_deep_dive.typ` を生成する

**必須8セクション**:
1. 技術ライフサイクルの位置づけ（Patent/Academic/Newsの3トレンド比較、現在のフェーズ判定）
2. 研究-実装タイムラグ分析（Academic→Patent→Newsの時間差、技術移転速度の評価）
3. マクロ環境イベントの影響分析（Policy/Marketイベントと特許トレンドの変曲点の照合）
4. **学術ランドスケープ分析**（`nebula_academic_clusters.json`がある場合は必須）:
   - 学術論文のクラスタ構造（何個の研究テーマクラスタが検出されたか、各クラスタのラベルと件数）
   - **超領域分析（Saturn Vと同様の高次構造分析）**: `spatial_context`の近接関係データを使い、近接するクラスタ群を「研究超領域」としてグルーピングする。Saturn Vの命名と同様に、学術側も命名する。各超領域に2-3段落の解説を付ける
   - **超領域間の橋渡し分析**: 異なる超領域間を接続する研究テーマ（クラスタ境界に位置するもの）を特定し、学際的研究の接点を記述する
   - 代表論文の引用（各クラスタの`representative_papers`から2-3件を具体的に引用: タイトル、著者/出典、年、研究の意義）
   - **特許クラスタ（Saturn V）との対比（P13パターン）**: 
     - 学術超領域とSaturn V超領域を対応させ、研究→特許化のパイプラインを評価
     - 学術で大きいが特許で小さい → 特許化の機会（提供された特許データセットの範囲に起因するカバレッジ制約の注記を付す。`database_name` が `voyager/context.json` の `population_meta` に指定されていればその名前を使用、未指定なら「提供された特許データセット」と汎用表記）
     - 特許で大きいが学術で小さい → 学術基盤が薄い、または実用化が先行
     - 両方で成長 → 最も有望な技術領域
5. **学術クラスタ動態マップ**（`cluster_dynamics`がある場合は必須）:
   - 4象限（成長リーダー/新興/成熟/ニッチ衰退）の各クラスタを解説
   - Saturn V特許クラスタの動態マップと比較し、研究と特許で成長段階が異なるテーマを特定
   - 「学術で新興 x 特許でニッチ/衰退」のテーマは将来のブレイクスルー候補として重点記述
6. 外部環境からの主要仮説（Mission Objectiveに関連する3-5個の仮説を環境データ + 学術ランドスケープから導出）
7. 環境分析サマリー
8. ミクロ分析A: マクロイベント・学術トレンドと対応する代表特許/論文（5-8件引用）

**最低120行**（学術ランドスケープ分析を含むため）。 `wc -l reports/nebula_deep_dive.typ` で確認する。

> **環境分析の位置づけ**: NEBULA deep_diveで導出した「外部環境からの主要仮説」は、Step 1-6の各deep_diveの冒頭で参照される。例えば「NEBULA環境分析で特定された2020年の規制変更が、Saturn Vクラスタ構造にどう反映されているか」といった文脈で、環境→特許の因果関係を分析する。

### モード `web_compensation` の場合（Web 調査で補完）

NEBULA データが存在しないため、Phase B の Web 調査結果を元に `reports/external_env_deep_dive.typ` を生成する。

**ファイル名**: `external_env_deep_dive.typ`（`nebula_deep_dive.typ` ではないので注意）

**必須 4 カテゴリ**（Phase A STOP-GATE D で確定した `web_coverage_categories` と一致）:

1. **市場規模・業界統計**
2. **政策・規制動向**
3. **学術動向・キーパーソン**
4. **主要企業動向**

**執筆要件**:

- 各カテゴリにつき 1-2 段落（各 5-8 行）
- 各主張に `#footnote[出所名, URL, 取得日]` を最低 1 件付与
- 付録 C の Web 調査出所一覧と整合させる
- 最低 80 行（NEBULA deep_dive の 120 行より短めでも可、ただし 4 カテゴリ網羅が必須）

**記載テンプレート**:
```typst
#import "report_style.typ": *

= 外部環境分析（Web 調査に基づく）

{4 カテゴリで得た外部情報と本母集団の特許動向を対比し、整合点・乖離点を
簡潔にまとめる（10-15 行）。NEBULA の「学術-特許クロス分析」（P13）の代替となる位置付け}
```

**Step 1-6 の各 deep_dive の冒頭で参照**: モード `execute` と同様に、`external_env_deep_dive.typ` で導出した外部環境仮説を各 Step で参照する。

### モード `omit` の場合（省略）

**Step 0 自体をスキップ**:
- `reports/nebula_deep_dive.typ` も `external_env_deep_dive.typ` も作らない
- Step 1-6 の各 deep_dive で「NEBULA 環境分析」「外部環境からの仮説」といった記述は省略
- 仮説検証サマリーでは NEBULA 由来仮説を含めない（特許データ由来の仮説のみ）

## Step 1: Saturn V deep_dive

Saturn V / EAGLEデータがある場合は必須。

**手順**:
1. `capcom_schema/exemplars/saturnv_exemplar.typ` を読む — **exemplarを読まずに書き始めてはならない**。コメント内の `required_sections` を確認する
2. Evidence JSON（ev*_saturn_v.json）+ `prompts/saturn_main_insight.md` を読む
3. **`data/saturnv_clusters.json` を必ず読む** — `noise_analysis` と `cluster_dynamics` と `spatial_context` を確認する
4. `reports/saturnv_deep_dive.typ` を生成する

**必須13セクション**:
1. 全体構造の概要（ノイズ率の戦略的解釈含む。`analysis/noise_analysis.md` の5手法を適用）
2. クラスタ規模の階層構造（メガ/ミドル/マイクロ3層）
3. UMAP空間の超領域分析（座標付き、命名、各超領域に2-3段落）
4. 超領域間ブリッジの戦略的分析（最低3つ）
5. ホワイトスペース分析（最低2つの空白地帯）
6. バリューチェーン分析（最低2つの技術連鎖）
7. 競争構造分析（日本vs海外の超領域別支配構造）
8. **クラスタ動態マップ分析**（`cluster_dynamics`がある場合は必須）:
   - `saturnv_clusters.json` の `cluster_dynamics` を読み、4象限にプロットされた全クラスタを解説する
   - **成長リーダー**（右上）: 大規模x高成長クラスタの技術的意味と戦略的重要性
   - **新興クラスタ**（左上）: 小規模x高成長クラスタ → 今後主流になる可能性がある技術テーマ
   - **成熟クラスタ**（右下）: 大規模x低成長 → 差別化が困難になっている領域
   - **ニッチ/衰退**（左下）: 撤退検討 or ニッチ戦略の深掘り
   - 4象限の分布バランスから、技術分野全体の成熟度を判定する
9. **ノイズ萌芽技術の詳細分析**（`noise_analysis`がある場合は必須）:
   - ノイズ率の解釈（`noise_interpretation`）
   - 時系列パターン（`temporal_pattern`）: 近年集中なら新興テーマ
   - 出願人集中度（`top_applicants`）: 集中なら戦略ニッチ
   - 萌芽キーワード（`emerging_keywords`）から具体的なテーマ名を2-3個抽出・命名
10. 統合的戦略インサイト（3つ以上、命名付き）
11. クロスモジュール検証
12. ミクロ分析A: 超領域別の代表特許（各超領域から2-3件、合計15件以上）
13. ミクロ分析B: 主要出願人の技術戦略プロファイル（5社以上）

**最低250行**。 `wc -l reports/saturnv_deep_dive.typ` で確認する。

## 1. 特許データとNPLデータの非対称性

APOLLO は提供された特許データセットをそのまま分析対象とするため、**ユーザーが用意した特許データベースのカバレッジに分析結果が強く依存する**。ギャップ分析（特許 vs NPL）の際、以下の構造的バイアスを考慮すること。

なお、ユーザーが CAPCOM ページで「使用した特許データベース名」を入力していれば、`voyager/context.json` の `population_meta.database_name` に値が入っているため、本文の記述をその名前で具体化すること（未指定なら「提供された特許データセット」と汎用表記）:

| データ種別 | カバレッジ特性 | 典型的な偏り |
|-----------|--------------|-----------|
| 特許（提供データセット） | **提供元データベースの仕様に依存** | カバー範囲（国・言語・出願種別）がユーザー側データベース仕様に依存する。国内特許のみのデータベースを使った場合、米国・欧州・中国の出願が含まれないため、グローバルな技術動向の一部しか捕捉できない |
| 学術論文 (Academic NPL) | **英語が主体**（OpenAlex 等） | グローバルな研究動向を広く捕捉。地域限定の特許データよりも件数・範囲ともに大きくなりやすい |
| ニュース (Business NPL) | **言語依存** | 日本語ニュースのみの場合、海外メディアの報道がカバーされず件数が少なくなる |
| 政策 (Policy NPL) | **地域依存** | 特定国の政策のみの場合がある |

## 2. 母集団スコープの明示（全章共通・絶対遵守）

### 2-0. 原則: 「本母集団」と「業界全体」は別物

APOLLO が分析する母集団は **ユーザーが検索式で絞り込んだ特許の部分集合** であり、**業界全体・市場全体・技術分野全体** とは一致しない。以下はギャップ分析章に限らず **レポート全ての章で守るべき原則**：

- 本母集団の観察は、業界全体の傾向を直接は示さない
- 「本母集団の上位」「本母集団の増加率」「本母集団における集中度」を「業界の上位」「市場成長率」「業界構造」と等号でつないではならない
- 業界全体への一般化が必要な場合は、Web 調査で外部データを `#footnote` で引用するか、母集団のカバレッジを論証する（詳細: `terminology.md` §6）

### 2-1. 各章でのスコープ明示ルール

| 章 | スコープ明示の要求水準 |
|---|---|
| 本分析の前提 | 「本分析の範囲と限界」サブセクションを必ず書き、母集団と業界全体の差分を明示（→ `report_structure.md` §3） |
| エグゼクティブサマリー | 冒頭 1 行で対象範囲を明示。以降の主張は母集団限定表現で統一 |
| 各モジュール deep_dive | 章冒頭または最初の evidence-box で「本母集団において〜」等を最低 1 回明示。ランキング・成長率・集中度の主張には必ず母集団限定修飾を付す |
| クロスモジュール分析 | 各「結論」段で母集団限定を再確認。業界全体への一般化を含む場合は外部裏付けを添える |
| 戦略的提言 | 提言が母集団内観察に基づくか、外部情報を加味した一般化かを区別 |
| 別冊（経営層向け要約版） | 特に厳格。エグゼクティブサマリー冒頭 + 各「発見」でスコープ明示（→ `executive_summary_guide.md` §3-6） |

### 2-2. ギャップ分析（特許 vs NPL）時の追加注意

1. **「論文は多いが特許が少ない」= 即座に「特許化の遅れ」ではない**: 地域限定特許データと全世界英語論文を比較している場合、見かけ上のギャップが生じる。この場合「提供された特許データの範囲では特許化が確認されていない」と記述し、「特許化が遅れている」と断定しないこと
2. **件数の直接比較は避ける**: 特許500件 vs 論文2,000件の件数差は、カバレッジの違いに起因する可能性が高い。件数比ではなく**トレンドの方向性**（成長/停滞/衰退）で比較する
3. **ハイプサイクルの解釈**: 特許の棒グラフ（主軸）と論文の折れ線（副軸）を比較する際、スケールの違いに注意。同一軸で重ねていないため、絶対値ではなく**ピーク時期のずれ**（タイムラグ）に注目する
4. **レポートでの記載方法**: ギャップ分析の結論には必ず、以下のいずれかに相当する注記を付すこと:
   - `database_name` が指定されている場合: 「なお、本分析の特許データは {database_name} から抽出された範囲に限定されており、グローバルな特許動向を完全に反映するものではない」
   - `database_name` が未指定の場合: 「なお、本分析の特許データは提供された特許データセットの範囲に限定されており、その収録範囲外の出願は含まれない」

   **特定のデータベース名（J-PlatPat 等の具体名）を執筆者が勝手に補うことは禁止**。`database_name` フィールドが空なら汎用表記を使うこと

### 2-3. NG / OK 例（スコープ明示）

❌ **NG（母集団と業界全体を混同）**:
> ギャップ分析の結果、業界では学術研究が先行し特許化が遅れている。企業は急いで特許化を進めるべきである。

✅ **OK（母集団限定 + 限界の明示）**:
> 本母集団では、NEBULA 学術クラスタの出現ピーク（2019 年）と特許出願ピーク（2022 年）の間に約 3 年のタイムラグが観察される。ただし学術論文は英語媒体のグローバルデータであるのに対し、本特許データは {database_name} から抽出された範囲に限定されており、両者のカバレッジは非対称である。従って「グローバル学術 vs 本母集団特許」という非対称な比較であることを明示した上で、本母集団の範囲では直近 3 年の出願加速が観察されていると結論づける。

## 3. Web調査のルール

### 調査の目的
特許データだけでは得られない外部コンテキスト（市場動向・企業戦略・政策・学術トレンド等）をWeb調査で積極的に収集し、分析の深みと説得力を高める。**NEBULAデータの有無にかかわらず、外部情報の補完を常に検討すること。**

### ユーザー確認（Phase B完了前に実施）
「分析の補強のためにWeb調査を行います。以下のテーマで検索してもよいですか？」と、調査予定のテーマ3-5件を提示して確認を得る。

### 積極的に調査すべき場面
- 主要出願人（上位5社程度）の事業戦略・M&A・アライアンス・プレスリリース
- 分析対象の技術分野の市場規模・成長予測（調査会社レポートの概要）
- 関連する政策・規制動向・標準化活動（政府機関の公表情報）
- ノイズ（萌芽技術）の実用化動向・スタートアップ情報
- 学術論文の引用動向やキーパーソンの研究活動
- 競合他社の特許以外の技術開発動向（製品発表、技術ブログ等）

### 調査の進め方
- 1テーマにつき2-3回の検索を行い、信頼性の高いソース（政府機関、調査会社、企業IR、学術機関）を優先する
- 得られた情報は即座にdeep_diveの該当セクションに反映する
- **APOLLO分析で見えたトレンド（例: クラスタ動態で「新興」と判定された技術）をWebで裏付ける**ことで、データドリブンな仮説がファクトベースの結論に昇格する

### Web情報の出所記載ルール（⚠️ 厳守 — 違反は品質不合格、v8 で引用形式を一本化）

**Web調査で得た情報をレポートに1文字でも使用する場合、出所情報は必ず「ページ下部の脚注（footnote）」として記載する。本文中に "_出所: ... _" のような平文の出所記述を書くのは禁止。**

#### 出所記載の必須3要素
| 要素 | 必須 | 例 |
|------|------|-----|
| **サイト名/発行元** | ✅ 必須 | 「経済産業省」「日本経済新聞」「Grand View Research」|
| **URL** | ✅ 必須 | https://www.meti.go.jp/xxx |
| **取得日** | ✅ 必須 | 取得日: 2026-04-14 |

#### 標準形式: `#footnote[...]` でページフッターへ（唯一の本文中引用方式）

Typst の脚注機能 `#footnote[...]` をすべての本文中 Web 引用に使う。脚注は**該当ページの下部**にページ連番で自動配置され、学術論文スタイルの引用形式になる。

```typst
世界CNF市場はCAGR 17.5-24.7%で成長し、2035年に64億ドルに達すると予測されている
#footnote[Grand View Research "Cellulose Nanofiber Market Report" (https://www.grandviewresearch.com/xxx), 取得日: 2026-04-14]。
```

**evidence-box 内で Web 情報を使う場合も同じ脚注形式を徹底する**（本文中に "_出所: ... _" を書かない）:

```typst
#evidence-box(99, "Web調査: 旭化成のCNF事業戦略")[
  旭化成は2024年にCNFゴムマスターバッチの量産設備を稼働させ、タイヤメーカーへの供給を本格化している#footnote[「旭化成 IR資料 2024年度」(https://www.asahi-kasei.com/ir/xxx), 取得日: 2026-04-14]#footnote[「日刊工業新聞」(https://www.nikkan.co.jp/xxx), 取得日: 2026-04-14]。Saturn V TELESCOPE 分析のクラスタ動態マップにおけるクラスタ 0「タイヤ用ゴム」の CAGR +254.5% は、この事業展開と整合する。
]
```

複数の出所を並記する場合は、`#footnote[...]#footnote[...]` のように複数の脚注を連続して付ける（1 つの脚注内に複数出所を詰め込まない）。

#### 付録での一括出所リスト（補助資料）

脚注による本文中引用に加え、レポート末尾の付録に **「付録 C: Web 調査出所一覧」** として番号付き一覧表を配置する。脚注と付録は**内容が一致**すること（脚注 No. = 付録 No. と対応するのが理想）。

```typst
== C. Web調査出所一覧

#styled-table(
  columns: (auto, 3fr, 2fr, auto),
  header: ("No.", "情報内容", "出所", "取得日"),
  [W1], [世界CNF市場予測], [Grand View Research (URL)], [2026-04-14],
  [W2], [旭化成CNF事業戦略], [旭化成IR資料 (URL)], [2026-04-14],
  [W3], [食品添加物公定書改定], [厚生労働省 (URL)], [2026-04-14],
)
```

#### 品質検証時のチェック

Phase D の品質検証で以下を確認すること:
```bash
# footnote による引用形式を使っているか
grep -c "#footnote\[" reports/report.typ

# 本文中に旧形式の "_出所:" が残っていないか（0 が合格）
grep -cE '_出所:|_Source:' reports/report.typ
```

**不合格パターン（絶対に避けること）:**
- 「世界CNF市場はCAGR 24.7%で成長する見込みである。」← 出所なし
- 「旭化成は2024年にCNF量産設備を稼働させた。」← 出所なし
- 「...設備を稼働させた_出所: 日刊工業新聞 (URL), 取得日: 2026-04-14_。」← 本文中インライン italic 出所（本方式では禁止）

**合格パターン:**
- 「世界CNF市場はCAGR 24.7%で成長する見込みである#footnote[Grand View Research (URL), 取得日: 2026-04-14]。」
- 「旭化成は2024年にCNF量産設備を稼働させた#footnote[日刊工業新聞 (URL), 取得日: 2026-04-14]。」

### その他のルール
- Web情報はAPOLLO分析データと**対比・照合**して使う。データ→外部情報→結論の論理チェーンを構築する
- Web情報のみに基づく主張は不可。必ずAPOLLOデータの裏付けと組み合わせる
- ユーザーが「不要」と回答した場合はセッション内データのみで分析を完結させる

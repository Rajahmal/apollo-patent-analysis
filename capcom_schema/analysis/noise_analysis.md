# ノイズ分析フレームワーク

## 概要

HDBSCANがクラスタに割り当てなかった特許（cluster == -1）は「ノイズ」と呼ばれる。
ノイズは分析の残滓ではなく、**新興技術や境界領域技術の宝庫**である。

`saturnv_clusters.json` の `noise_patents` 配列（最大200件）を使用して分析する。
全ノイズ特許の完全データは `patents.csv` で `cluster == -1` でフィルタ可能。

---

## 分析手法

### 1. ノイズ率の戦略的解釈

`metadata.noise_ratio` を確認:

| ノイズ率 | 解釈 | 戦略的意味 |
|---------|------|----------|
| 5%以下 | 技術分野が高度に均質 | 成熟市場。差別化困難 |
| 5-15% | 標準的 | 安定的な技術構造。一部の新興テーマが存在 |
| 15-30% | 多様性が高い | 技術の分岐・融合が活発。イノベーション機会が豊富 |
| 30%超 | 発散的 | 多くの未分類テーマ。黎明期の技術分野か、分析パラメータの見直しが必要 |

**記述パターン**: 「ノイズ率{ratio}%（{noise_count}件/{total_patents}件）は{解釈}を示しており、{戦略的意味}を示唆する」

### 2. 空間分布分析

`noise_patents` の `umap_x`, `umap_y` 座標を `clusters` の `centroid` と比較:

- **特定エリアに集中** → 既存クラスタに近い新興テーマの候補
  - 最も近いクラスタを特定し、そのクラスタの拡張技術と解釈
- **クラスタ境界に散在** → ブリッジ技術（複数領域にまたがる融合技術）
  - 2つ以上のクラスタの中間にあるノイズは技術融合の先駆
- **空白地帯に孤立** → 完全に新しい技術領域
  - 既存クラスタから最も離れたノイズは最も革新的な可能性

**分析手順**:
1. `noise_patents` の座標を確認
2. 各クラスタの `centroid` からの距離を概算
3. 近接パターン（集中/散在/孤立）を判定
4. 近接クラスタとの関連を解釈

### 3. 時系列分析

`noise_patents` の `year` を集計:

- **直近年に増加** → 新興技術の萌芽。次回分析でクラスタ化される可能性大
- **過去に集中** → 成熟技術のロングテール（標準化済みバリエーション）
- **通期で均等** → 常に一定割合の非主流技術が存在

**記述パターン**: 「直近3年のノイズ特許は{N}件で{増加/減少}傾向にあり、{年}以降に出現した新テーマが含まれている」

### 4. 出願人分析

`noise_patents` の `applicant` を集計:

- **特定企業に集中** → その企業の独自技術戦略（ニッチ・先行投資）
- **多数の企業に分散** → 技術の標準化前段階（各社が異なるアプローチを模索）

**記述パターン**: 「{企業名}はノイズ特許{N}件中{M}件を占め、既存クラスタに分類されない独自技術の開発を進めている」

### 5. ノイズからの新興領域抽出

**最も重要な分析**。ノイズ特許のタイトルを確認し、共通テーマを手動で抽出:

1. `noise_patents` の `title` を読む
2. 類似タイトルのグループを特定（2-3件以上の共通テーマ）
3. 各グループの `umap_x`, `umap_y` 座標の近接性を確認
4. 年・出願人との交差分析
5. 将来的にクラスタ化される「萌芽領域」として報告

**記述パターン**: 「ノイズ{N}件中、{テーマキーワード}に関連する{M}件が座標(x, y)付近に集中しており、既存の{近接クラスタ名}とは異なる新技術領域を形成しつつある」

## ノイズ分析のレポート記述ガイド

### 推奨構成

```
=== ノイズ分析: 新興技術の萌芽

1. **概況**: ノイズ率とその解釈
2. **空間パターン**: UMAP上の分布特徴
3. **萌芽領域の特定**: タイトル分析から抽出した新興テーマ（2-3件）
4. **戦略的含意**: ノイズから読み取れる技術動向の変化
```

### 4層モデルでの記述

- Layer 1（事実）: 「ノイズ率15.3%（95件/621件）」
- Layer 2（解釈）: 「多様性の高い技術構造を示唆」
- Layer 3（So What?）: 「{テーマ}に関する萌芽技術が形成されつつある」
- Layer 4（提言）: 「{テーマ}領域のウォッチリスト化を推奨」

## データソース

| データ | ファイル | アクセス方法 |
|--------|---------|-----------|
| ノイズ特許座標・基本情報 | `data/saturnv_clusters.json` → `noise_patents` | 最大200件。タイトル100文字制限 |

## クラスタ動態マップとのクロスリファレンス

ノイズ分析はクラスタ動態マップ（累積件数 × CAGR 4象限）と組み合わせることで、萌芽技術の成長軌跡をより精密に追跡できる。

### ノイズ → 新興クラスタへの成長パターン

過去の分析でノイズだった特許群が、データ追加やパラメータ調整により新興クラスタとして検出されるケースがある。このパターンを検出するために:

1. **現在のノイズ萌芽テーマ**（手法5で抽出）と**クラスタ動態マップの新興象限**（左上: 小規模×高成長）を照合する
2. 新興象限のクラスタが過去のノイズから成長した事例を特定し、「ノイズ→クラスタ成長」のリードタイムを推定する
3. 現在のノイズ萌芽テーマのうち、件数・成長率から次回分析でクラスタ化される可能性が高いものを予測する

### 記述パターン

「ノイズ萌芽テーマ{テーマ名}（{N}件）は、クラスタ動態マップの新興象限に位置するクラスタ{ID}と技術的に近接しており、次回分析でクラスタとして検出される可能性が{高い/ある}」

### ノイズ率とクラスタ動態の相関

| ノイズ率変化 | クラスタ動態マップの傾向 | 解釈 |
|------------|----------------------|------|
| ノイズ率上昇 + 新興クラスタ増加 | 技術分野が活発に拡張中 | イノベーション機会が豊富 |
| ノイズ率上昇 + 成熟クラスタ支配 | 既存技術からの逸脱が増加 | パラダイムシフトの兆候 |
| ノイズ率低下 + 成長リーダー安定 | 技術構造が収斂 | 標準化・寡占化が進行 |
| ノイズ率低下 + 衰退クラスタ増加 | 技術分野全体が成熟 | 差別化困難。隣接分野への展開が必要 |

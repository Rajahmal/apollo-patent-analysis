# クロスモジュール分析パターン

単一モジュールの分析だけでは「片面的な結論」になりがちである。複数モジュールのデータを組み合わせることで、仮説の検証、矛盾の発見、統合的な洞察が可能になる。

> **選択的読み込み**: 本ファイルは13種のクロスモジュール分析パターン（P1-P13）を含む。**全パターンを読む必要はない。** まず「基本原則」を読み、Mission Objectiveに関連するパターンのみを選択的に読むこと。
> - 技術動態分析 → P1, P4, P11
> - キーワード・技術構造 → P2, P5, P6
> - 人的ネットワーク → P3, P9
> - ルール vs AI分類 → P7
> - 環境分析（NEBULA環境コンテキスト）→ P8, P12
> - 学術-特許連携 → P13
> - 総合結論 → P10

---

## 基本原則

1. **仮説→検証のループ**: 1つのモジュールで得た結論を「仮説」とし、別のモジュールのデータで「検証」する
2. **最低3パターン**: 全ての主要な結論は最低3つのクロスモジュール分析パターンで裏付ける
3. **矛盾こそ宝**: モジュール間で矛盾するデータが見つかった場合、それは重要な発見である。矛盾を隠さず、なぜ矛盾が生じるかを考察する
4. **データソース明示**: クロス分析の結果を記述する際は、どのモジュールのデータを使ったかを明示する

## P1: 技術領域 x 動態 (Saturn V + MEGA)

### 目的
Saturn Vで分類された技術クラスタが、MEGA PULSEの4象限でどこに位置するかを照合し、各技術の成長段階を評価する。

### 手順
1. Saturn Vの各クラスタの主要出願人リストを確認する
2. MEGA PULSEの `entities` で同じ出願人を検索し、象限位置を確認する
3. 「クラスタXの主要出願人の大半がQIに位置する」= 成長技術、「QIIIに位置する」= 衰退技術、と評価する
4. クラスタ内で出願人の象限がバラバラな場合 → その技術領域は競争構造が分化している

### 文章化テンプレート

> Saturn Vクラスタ{X}（{技術名}, {N}件）の主要出願人{A社, B社, C社}をMEGA PULSEで確認したところ、{A社}はQI（CAGR +{Y}%）、{B社}はQIV（CAGR {Z}%）に位置した。これは同クラスタ内で{解釈}を示しており、{So What}と考えられる。

## P2: キーワード構造 x 技術領域 (Explorer + Saturn V)

### 目的
Explorerの共起ネットワークで検出された技術キーワードのクラスタが、Saturn Vの特許クラスタとどう対応するかを照合する。

### 手順
1. Explorerのコミュニティ上位キーワードを確認する
2. Saturn Vの各クラスタのラベル・TF-IDF上位語と比較する
3. 「Explorerコミュニティ1のキーワード群はSaturn Vクラスタ3に対応」等のマッピングを作成する
4. 対応しないキーワード（ExplorerにあるがSaturn Vクラスタに該当なし）= 横断的技術概念

### 文章化テンプレート

> Explorerの共起ネットワークで最大コミュニティを形成する「{キーワード群}」は、Saturn Vクラスタ{X}（{技術名}）と高い対応を示す。ただし、「{キーワードA}」はクラスタ{X}と{Y}の両方に出現し、技術横断的な概念であることが示唆される。

---

## P3: 人的ネットワーク x 技術領域 (CREW + Saturn V)

### 目的
CREW発明者/出願人ネットワークの構造が、Saturn Vの技術クラスタ構造とどう対応するかを照合し、「誰が」「何を」研究しているかの全体像を構築する。

### 手順
1. CREWのコミュニティ別メンバー（所属組織・中心人物）を確認する
2. Saturn Vクラスタの主要出願人リストと照合する
3. CREWコミュニティ = Saturn Vクラスタの場合 → 技術と組織が対応した明確な構造
4. CREWコミュニティが複数Saturn Vクラスタにまたがる場合 → 多角的な研究体制
5. ブローカー人物のクラスタ所属を確認 → 技術融合のキーパーソン

### 文章化テンプレート

> CREWネットワークのコミュニティ{A}（中心人物: {人物名}, {N}名）は、Saturn Vクラスタ{X}（{技術名}）に集中して出願している。一方、ブローカーの{人物名}（betweenness {値}）はクラスタ{X}と{Y}の両方に関与しており、{技術融合の解釈}。

## P4: 市場構造 x 競争環境 (ATLAS + MEGA)

### 目的
ATLASの基本統計（出願人ランキング、HHI、時系列）とMEGAの動態分析を照合し、市場構造の変化を評価する。

### 手順
1. ATLASの出願人ランキング（件数）とHHIを確認する
2. MEGA PULSEの各象限のプレイヤー構成と比較する
3. ATLASで上位だがMEGAでQIII/QIV → 「かつての王者」が衰退中
4. ATLASで下位だがMEGAでQI/QII → 新興勢力の台頭
5. HHI推移とMEGA象限分布の変化を時系列で照合する

### 文章化テンプレート

> ATLASの出願件数ランキングでは{A社}が{N}件でトップだが、MEGA PULSEではQIV（CAGR {X}%）に位置し成長が鈍化している。一方、ランキング{M}位の{B社}（{K}件）はQI（CAGR +{Y}%）に位置し、{今後の展望}。

## P5: 競争比較 x 技術ポートフォリオ (Explorer Dominance + Saturn V)

### 目的
Explorer Dominanceの出願人別キーワード優位性とSaturn Vの技術クラスタを照合し、各プレイヤーの技術ポートフォリオの強弱を評価する。

### 手順
1. Explorer Dominanceの出願人別優位キーワードを確認する
2. 各キーワードがSaturn Vのどのクラスタに該当するかをマッピングする
3. 「A社はクラスタ1・3で優位、B社はクラスタ2・5で優位」等のポートフォリオ比較を作成する
4. 複数クラスタで優位な出願人 → 広範な技術ポートフォリオ
5. 特定クラスタのみで優位 → 専門特化型の戦略

### 文章化テンプレート

> Explorer Dominance分析によると、{A社}は「{キーワード群}」で高い優位性を持ち、これはSaturn Vクラスタ{X, Y}に対応する。{B社}は「{キーワード群}」で優位であり、クラスタ{Z}に特化した戦略を採っている。{両社のポートフォリオ比較からの示唆}。

## P6: トレンドキーワード x 動態 (Explorer Trend + MEGA)

### 目的
Explorer Trendで検出された急上昇キーワードの関連出願人がMEGAの4象限のどこに位置するかを照合し、新興トレンドの担い手を特定する。

### 手順
1. Explorer Trendの急上昇キーワードを確認する
2. 急上昇キーワードを含む出願の出願人をpatents.csvで検索する（ミクロ分析）
3. 当該出願人のMEGA PULSE象限位置を確認する
4. QI/QIIの出願人がトレンドを牽引 → 成長テーマとプレイヤーの一致
5. QIII/QIVの出願人がトレンドに乗る → 事業転換の試みの可能性

### 文章化テンプレート

> Explorerで急上昇キーワード「{KW}」が検出された。ミクロ分析によると、当該キーワードを含む特許{N}件のうち{A社}が{M}件を占める。MEGA PULSEで{A社}はQI（CAGR +{X}%）に位置しており、同社の成長を牽引するテーマの一つであると考えられる。

## P7: ルール分類 x AI分類 (CORE + Saturn V)

### 目的
COREのルールベース分類とSaturn VのAIクラスタリングを照合し、人間の分類仮説とAIの分類結果の一致・乖離を評価する。

### 手順
1. COREの分類カテゴリとSaturn Vのクラスタを対応づける
2. 一致 → 分類仮説がデータで裏付けられた
3. 乖離 → AIが人間の想定外のグルーピングを発見した可能性
4. COREで未分類だがSaturn Vでクラスタ化されたもの → 新しい分類カテゴリの候補

### 文章化テンプレート

> COREのルールベース分類「{カテゴリ名}」（{N}件）は、Saturn Vクラスタ{X}（{M}件）と{一致率}%の対応を示した。乖離分の{K}件はクラスタ{Y}に分類されており、{解釈: ルール定義の見直しが必要 / AIが新しいサブカテゴリを発見}。

## P8: 環境分析 x 技術動態 (NEBULA + ATLAS + MEGA + 全モジュール)

### 目的
NEBULAの非特許文献（論文・ニュース・政策）のトレンドとATLAS/MEGAの特許動態を照合し、外部環境と技術開発の関係を評価する。NEBULA環境分析はレポートの冒頭に配置され、後続の全モジュール分析の「外部環境コンテキスト」として機能する。

### 手順
1. NEBULAのハイプサイクル分析で各技術のフェーズを確認する
2. ATLASの出願時系列とNEBULAの論文・ニュース時系列を重ねる
3. 論文ピーク → 特許ピークのタイムラグを計測する
4. 政策イベント（法改正、補助金、規制）と出願急増の対応を確認する
5. MEGAの象限移動とNEBULAイベントの因果関係を推定する
6. NEBULA deep_diveで導出した「外部環境からの主要仮説」を一覧化する
7. **各モジュールのdeep_diveで仮説の支持/棄却を記述する**（環境コンテキスト参照）

### 環境コンテキスト参照パターン（v1.1）

NEBULA deep_diveが存在する場合、Step 1-6の各deep_diveの冒頭で以下を実施すること:
- NEBULA環境分析の主要仮説を参照し、そのモジュールの分析結果が仮説を支持/棄却するか論じる
- 例: 「NEBULA環境分析で特定された{年}年の{政策/市場変化}は、本モジュールの分析結果において{クラスタ構造の変化/出願人動態}として観測された」

### 文章化テンプレート

> NEBULAのハイプサイクル分析で「{技術名}」は{フェーズ}に位置する。ATLASの時系列と照合すると、論文ピーク（{年}年）から特許ピーク（{年}年）まで約{N}年のタイムラグがある。MEGA PULSEでの象限位置（{象限}）を考慮すると、{今後の見通し}。

> NEBULA環境分析で導出した仮説「{仮説}」について、Saturn Vクラスタ分析の結果は{支持/棄却/部分的支持}を示している。具体的には、{クラスタXの構造/出願人動態/キーワード出現パターン}が環境変化と{対応関係の詳細}。

## P9: 手動分類 x 自動分類 x 人的NW (EAGLE + Saturn V + CREW)

### 目的
EAGLEの手動クラスタリング、Saturn VのAIクラスタリング、CREWの人的ネットワークの3つを照合し、技術・人・組織の三位一体の分析を行う。

### 手順
1. EAGLEの手動クラスタとSaturn Vのクラスタの対応を確認する
2. 一致する部分 → 人間の直感とAIの分析が合致
3. EAGLEでは同一クラスタだがSaturn Vでは分離 → 人間が見落とした技術的差異
4. CREWの研究者コミュニティがEAGLE/Saturn Vのクラスタとどう対応するかを確認する
5. 技術クラスタと人的コミュニティが一致 → 組織的な研究体制が確立している

### 文章化テンプレート

> EAGLEの手動クラスタ「{名称}」は、Saturn Vクラスタ{X}と{Y}にまたがっている。Saturn Vが分離した理由として、ミクロ分析によると、クラスタ{X}は{技術A}、クラスタ{Y}は{技術B}に特化しており、人間の直感では1つだが技術的には2つの異なる方向性を含む。CREWでは{人物名}がこの2クラスタのブリッジ役を担っている。

## P10: 全体統合 (全モジュール) — Mission Objectiveへの総合回答

### 目的
Mission Objectiveへの最終回答として、全モジュールのデータを統合した結論を構築する。

### 手順
1. Mission Objectiveの問い（mission.json）を再確認する
2. 各P1-P9のクロス分析で得られた洞察を一覧化する
3. Mission Objectiveへの回答に直接貢献する洞察を選別する
4. 以下の構造で最終章（Conclusion / Strategic Recommendations）を構築する:
   - 全体サマリー: 1-2段落で分析全体の結論を述べる
   - 主要発見: 3-5項目の箇条書き（各項目に複数モジュールの根拠）
   - 戦略提言: 具体的なアクションアイテム（優先順位・時間軸付き）
   - リスク・留意点: 分析の限界・データの制約を正直に述べる

### 文章化テンプレート

> **⚠️ Mission Objective の原文を「{問い}」として転記しない。** エージェントが咀嚼し、「本分析が答えた問い」を自然な日本語で書き下すこと。
>
> 本分析では{N}モジュールのデータを横断的に分析し、{問いの咀嚼: 本分析が答えようとした内容を1行で書き下す}に対する回答を導出した。主要な発見は以下の通り:
>
> 1. **{発見1}**: {Saturn Vデータ} と {MEGAデータ} の照合から...
> 2. **{発見2}**: {Explorerデータ} と {CREWデータ} の照合から...
> 3. **{発見3}**: {ATLASデータ} と {NEBULAデータ} の照合から...
>
> 以上を踏まえ、以下を戦略提言として提案する:
> - **短期（1年以内）**: {具体的アクション}
> - **中期（1-3年）**: {具体的アクション}
> - **長期（3-5年）**: {具体的アクション}

## P11: クラスタ動態(Saturn V) x MEGA PULSE

### 目的
Saturn Vのクラスタ動態マップ（累積件数xCAGR）とMEGA PULSEの4象限（エンティティレベル）を対比する。
クラスタレベルの成長とエンティティレベルの成長が一致するか乖離するかを分析し、
「成長クラスタに属しているが衰退している出願人」や「衰退クラスタで急成長している新興プレイヤー」を特定する。

## P12: ノイズ分析(Saturn V) x 環境(NEBULA)

### 目的
Saturn Vのノイズ特許から抽出した萌芽テーマを、NEBULAの環境分析（学術論文トレンド、政策動向、市場動向）と照合する。
「学術論文では活発だがまだ特許のメインクラスタに成長していないテーマ」や
「政策変更を受けて今後ノイズからクラスタに成長しうるテーマ」を予測する。

## P13: 学術クラスタ(NEBULA) x 特許クラスタ(Saturn V)

### 目的
NEBULA学術ランドスケープのクラスタとSaturn V特許ランドスケープのクラスタを比較する。
研究→特許化のパイプラインを分析し、「研究は盛んだが特許化が遅れている領域」や
「特許は多いが学術的基盤が薄い領域」を特定する。

設計思想
01
二系統アーキテクチャ

APOLLOアプリ（Streamlit）はデータ処理・可視化に特化し、 レポート生成はClaudeCode（外部AI）に完全委譲する。 VOYAGERはGemini APIで「骨格10%」、 CAPCOMはClaudeCodeで「本格レポート90%」を担当する。

02
事前計算によるトークン効率

転換点検出・異常行動検出などの重い計算はPythonスクリプトが先行実行する。 ClaudeCodeはJSONを読むだけ（+~2,000トークン）。 WebSearchは既存フローに乗せ、追加コストをほぼゼロに抑える。

03
品質ゲートによる完了保証

各フェーズに機械的なゲートスクリプトを置き、 「AIの質的判断による上書き」を禁止する。 代表特許15件・出願人5社・最低行数はシェルスクリプトで客観判定される。

トークン消費内訳（参考）
処理 方式 追加トークン 備考
スクリプト実行（8本） bash run_scripts.sh 0 Python側で処理、AIは関与しない
JSON読み込み（転換点・異常行動） Read ツール ~2,000 事前計算済み結果を参照するだけ
Web調査（転換点裏取り） WebSearch（既存フロー） ~5,000–10,000 ユーザー確認後に実行。事前生成クエリを使用
Deep Dive執筆（Phase C） 生成 最大使用量 品質のためのトークン消費は許容
